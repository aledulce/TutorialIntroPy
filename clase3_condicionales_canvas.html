Apertura · 10 min
¿Qué es una estructura condicional?
Hasta ahora los programas ejecutan instrucciones de arriba hacia abajo, siempre en el mismo orden. Los condicionales permiten que el programa tome decisiones según los datos.
La lógica de decisión existe en todo sistema de gestión
Pensemos en cómo funciona la asignación de turnos en PAMI o ANSES. El sistema nunca hace lo mismo para todos: evalúa condiciones y actúa en consecuencia.
?
¿El ciudadano tiene cobertura activa?
Si sí → puede pedir turno. Si no → se muestra un mensaje de error.
?
¿El trámite requiere documentación adicional?
Si sí → se agrega un paso al formulario. Si no → se avanza directo.
?
¿Hay turnos disponibles hoy?
Si sí → se muestra el calendario. Si no → se ofrece el próximo día hábil.
Lo que vamos a aprender hoy
Al final de esta clase van a poder escribir programas que evalúan condiciones y ejecutan código diferente según el resultado.

Conceptos · 15 min
Booleanos y operadores de comparación
Antes de tomar una decisión, el programa necesita evaluar una condición. El resultado siempre es True o False.
El tipo bool
>>> type(True) <class 'bool'> >>> tiene_turno = True >>> sistema_activo = False >>> print(tiene_turno) True
En Python, True y False van con mayúscula inicial. true en minúscula da NameError.
Operadores de comparación
Comparan dos valores y devuelven un booleano:
>>> 10 == 10 # igual a True >>> 10 != 5 # distinto de True >>> 10 > 20 # mayor que False >>> 10 < 20 # menor que True >>> 10 >= 10 # mayor o igual True >>> 10 <= 9 # menor o igual False
Comparaciones con variables del sistema de turnos
turnos_disponibles = 12 edad_jubilado = 68 deuda_ciudadano = 0 >>> turnos_disponibles > 0 True >>> edad_jubilado >= 60 True >>> deuda_ciudadano == 0 True
Cada comparación genera un True o False. Esos son los valores que el if va a evaluar.
⚠️ El error más común: = vs ==
# = asigna un valor a una variable estado = "activo" # == compara dos valores >>> estado == "activo" True # Mezclarlos es uno de los errores más frecuentes >>> if estado = "activo": # SyntaxError File "<stdin>", line 1 SyntaxError: invalid syntax
Dentro de un if siempre se usa == para comparar, nunca =.
Conceptos · 20 min
La estructura if / else
La instrucción fundamental para tomar decisiones. Si la condición es verdadera, ejecuta un bloque. Si no, ejecuta el otro.
Sintaxis básica
if condicion: # bloque que se ejecuta si condicion es True instruccion_1 instruccion_2 else: # bloque que se ejecuta si condicion es False instruccion_alternativa
Dos reglas de sintaxis que no se pueden olvidar: los dos puntos : al final del if y el else, y la indentación de 4 espacios en cada bloque.
Python usa la indentación para definir bloques de código. No es una cuestión de estilo: si falta la indentación, el programa falla. La mayoría de los editores la agregan automáticamente.
1
Verificación de turno disponible
turnos_disponibles = 12 if turnos_disponibles > 0: print("Hay turnos disponibles. Puede continuar.") else: print("No hay turnos disponibles para hoy.") Hay turnos disponibles. Puede continuar.
Cambiar turnos_disponibles = 0 y ejecutar de nuevo. Observar cuál bloque se ejecuta.

Validación de cobertura
tiene_cobertura = True nombre_ciudadano = "Carlos Fernández" if tiene_cobertura: print(f"{nombre_ciudadano} tiene cobertura activa.") print("Puede solicitar turno médico.") else: print(f"{nombre_ciudadano} no tiene cobertura activa.") print("Debe regularizar su situación antes de continuar.") Carlos Fernández tiene cobertura activa. Puede solicitar turno médico.
Cuando la condición es una variable booleana, se puede escribir directamente if tiene_cobertura: en lugar de if tiene_cobertura == True:. Ambas formas son válidas, pero la primera es más pythónica.

El bloque else es opcional
# A veces solo queremos actuar si algo es True deuda = 0 if deuda > 0: print(f"El ciudadano tiene una deuda de ${deuda}.") print("Se envía notificación de pago.") # Si deuda es 0, no pasa nada y el programa sigue print("Verificación completada.") Verificación completada.
Conceptos · 20 min
elif — más de dos caminos posibles
Cuando hay más de dos posibilidades, elif (abreviatura de "else if") permite encadenar condiciones.
Sintaxis con elif
if condicion_1: # se ejecuta si condicion_1 es True elif condicion_2: # se ejecuta si condicion_1 es False y condicion_2 es True elif condicion_3: # se ejecuta si las anteriores son False y condicion_3 es True else: # se ejecuta si ninguna condición anterior fue True
Python evalúa las condiciones de arriba hacia abajo y ejecuta solo el primer bloque verdadero. Una vez que encuentra uno, ignora el resto.
1
Clasificación de trámites por prioridad
tipo_tramite = "urgente" if tipo_tramite == "urgente": print("Prioridad ALTA — ventanilla preferencial.") elif tipo_tramite == "normal": print("Prioridad NORMAL — espera estimada: 20 minutos.") elif tipo_tramite == "informativo": print("Prioridad BAJA — puede realizarse online.") else: print("Tipo de trámite no reconocido.") Prioridad ALTA — ventanilla preferencial.
2
Cálculo de descuento por categoría
Un sistema de tasas municipales aplica descuentos según la categoría del contribuyente:
categoria = "jubilado" tasa_base = 5000 if categoria == "jubilado": descuento = 0.50 elif categoria == "discapacidad": descuento = 0.75 elif categoria == "monotributista": descuento = 0.20 else: descuento = 0.0 monto_final = tasa_base * (1 - descuento) print(f"Categoría: {categoria}") print(f"Descuento aplicado: {int(descuento * 100)}%") print(f"Monto a pagar: ${monto_final}") Categoría: jubilado Descuento aplicado: 50% Monto a pagar: $2500.0
Probar con categoria = "discapacidad" y categoria = "empresa". Observar cómo cambia el resultado sin tocar el resto del código.
3
Condicionales anidados
Un if puede contener otro if adentro:
tiene_dni = True dni_vigente = True if tiene_dni: if dni_vigente: print("Identidad verificada. Acceso permitido.") else: print("DNI vencido. Debe renovarlo en el Registro Civil.") else: print("Sin documento de identidad. No se puede continuar.") Identidad verificada. Acceso permitido.
Con los anidamientos es fácil perder la indentación. Si el código empieza a tener muchos niveles, conviene reescribirlo usando operadores lógicos (siguiente sección).
Conceptos · 15 min
Operadores lógicos: and, or, not
Permiten combinar varias condiciones en una sola expresión, haciendo el código más legible y menos anidado.
and
ambas deben ser True
El resultado es True solo si todas las condiciones son verdaderas.
True and True → True
True and False → False
False and True → False
or
al menos una debe ser True
El resultado es True si al menos una condición es verdadera.
True or False → True
False or True → True
False or False → False
not
invierte el valor
Devuelve el valor opuesto: convierte True en False y viceversa.
not True → False
not False → True
Orden de evaluación
not → and → or
Python evalúa primero not, luego and y finalmente or. Los paréntesis cambian el orden.
Usar paréntesis para dejar claro el orden cuando hay dudas.
1
and — reemplazando un if anidado
# Versión con anidamiento (más difícil de leer) if tiene_dni: if dni_vigente: print("Acceso permitido.") # Versión con and (más legible) if tiene_dni and dni_vigente: print("Acceso permitido.")
Ambas versiones hacen exactamente lo mismo. La segunda se entiende de una sola lectura.
2
or — múltiples condiciones habilitantes
tipo_beneficio = "pensión" if tipo_beneficio == "jubilación" or tipo_beneficio == "pensión": print("Beneficiario ANSES — descuento del 50% aplicado.") else: print("Sin descuento aplicable.") Beneficiario ANSES — descuento del 50% aplicado.
3
Condición compleja real
Un sistema de AFIP habilita el plan de pagos si el contribuyente no tiene deuda previa y el monto supera cierto mínimo:
deuda_anterior = 0 monto_declarado = 85000 monto_minimo_plan = 10000 if deuda_anterior == 0 and monto_declarado >= monto_minimo_plan: print("Habilitado para plan de pagos en hasta 12 cuotas.") elif deuda_anterior > 0: print("No habilitado: regularizar deuda anterior primero.") else: print("Monto insuficiente para plan de pagos.") Habilitado para plan de pagos en hasta 12 cuotas.

Errores frecuentes en condicionales
Los errores más comunes al trabajar con estructuras condicionales y cómo resolverlos.
IndentationError: expected an indented block
Falta la indentación de 4 espacios después del if o el else. En Python la indentación no es opcional: define la estructura del código. VS Code la agrega automáticamente si se escriben los dos puntos y se presiona Enter.
SyntaxError al usar = en vez de == dentro del if
if estado = "activo": es un error de sintaxis. Para comparar siempre se usa ==. El operador = solo sirve para asignar valores a variables.
El else nunca se ejecuta aunque la condición debería ser False
Verificar que el else esté al mismo nivel de indentación que el if al que pertenece. Si está más adentro, Python lo interpreta como parte de otro bloque.
La condición siempre es True aunque el dato cambia
Revisar si se compara el valor correcto. if nombre == "ana": no es True si nombre = "Ana" — Python distingue mayúsculas. Usar nombre.lower() == "ana" para comparar sin importar el caso.
TypeError al comparar tipos incompatibles
if edad > "18": da error porque compara un número con un texto. Asegurarse de que ambos lados de la comparación sean del mismo tipo. Si viene de input del usuario, convertir con int().
