Apertura · 10 min
¿Para qué sirven los bucles?
En las clases anteriores aprendimos a tomar decisiones con if/else. Hoy aprendemos a repetir acciones automáticamente. Eso es exactamente lo que hacen los bucles.
El problema que resuelven
Imaginá que tenés que procesar 500 solicitudes de turnos pendientes en ANSES. Sin bucles, habría que escribir 500 líneas casi iguales. Con un bucle, se escribe la lógica una sola vez y Python la repite las veces necesarias.

Los bucles son la herramienta que permite que las computadoras hagan lo que mejor saben hacer: ejecutar tareas repetitivas sin cansarse ni equivocarse.
Los dos tipos de bucle que vamos a ver hoy
Bucle for
Iteración definida
Se usa cuando se sabe de antemano cuántas veces hay que repetir algo, o cuando hay que recorrer una colección de elementos.
Ej: Procesar cada solicitud en una lista de 200 expedientes pendientes.
Bucle while
Iteración condicional
Se usa cuando no se sabe cuántas repeticiones van a ser necesarias: el bucle continúa mientras se cumpla una condición.
Ej: Seguir pidiendo el número de DNI hasta que el usuario ingrese uno válido.

Concepto · 25 min
El bucle for
El bucle for recorre una secuencia de elementos, uno por uno, y ejecuta el mismo bloque de código para cada uno.
Estructura básica
for elemento in secuencia: # este bloque se ejecuta una vez por cada elemento print(elemento)
Dos reglas que siempre se cumplen: los dos puntos (:) al final de la línea del for, y la indentación (4 espacios o 1 Tab) en el bloque interior.
La indentación no es decorativa en Python: define qué líneas pertenecen al bucle. Un error de indentación es un error de lógica.
1
Recorrer una lista de trámites
tramites = ["Jubilación", "Asignación Universal", "Desempleo", "Pensión"] for tramite in tramites: print(f"Procesando trámite: {tramite}") Procesando trámite: Jubilación Procesando trámite: Asignación Universal Procesando trámite: Desempleo Procesando trámite: Pensión
La variable tramite toma un valor diferente en cada vuelta. Nosotros elegimos su nombre: podría llamarse t, item o cualquier cosa, pero un nombre descriptivo hace el código más legible.
2
Iterar con range()
range(n) genera la secuencia de números enteros del 0 al n-1. Es la forma más común de repetir algo una cantidad fija de veces.
# Generar 5 números de expediente for i in range(5): print(f"EXP-2024-{i:04d}") EXP-2024-0000 EXP-2024-0001 EXP-2024-0002 EXP-2024-0003 EXP-2024-0004
range() acepta hasta 3 argumentos: range(inicio, fin, paso). Por ejemplo, range(1, 11) da del 1 al 10; range(0, 100, 10) da 0, 10, 20 ... 90.
3
Acumular resultados dentro del bucle
Una variable definida antes del bucle puede actualizarse en cada vuelta para acumular un total.
montos = [15000, 42300, 8700, 31200, 19500] total = 0 for monto in montos: total = total + monto print(f" Sumando ${monto}... subtotal: ${total}") print(f"\nTotal de subsidios a liquidar: ${total}") Sumando $15000... subtotal: $15000 Sumando $42300... subtotal: $57300 Sumando $8700... subtotal: $66000 Sumando $31200... subtotal: $97200 Sumando $19500... subtotal: $116700 Total de subsidios a liquidar: $116700
Este patrón —inicializar una variable en 0 antes del bucle y sumarle valores adentro— es uno de los más usados en procesamiento de datos.
4
Combinar for con if
Los bucles y los condicionales funcionan juntos. La indentación define qué pertenece a qué.
solicitudes = [ {"nombre": "Ana García", "estado": "pendiente"}, {"nombre": "Luis Romero", "estado": "aprobada"}, {"nombre": "María López", "estado": "pendiente"}, {"nombre": "Carlos Díaz", "estado": "rechazada"}, ] for solicitud in solicitudes: if solicitud["estado"] == "pendiente": print(f"⚠ ATENCIÓN REQUERIDA: {solicitud['nombre']}") else: print(f" OK: {solicitud['nombre']} — {solicitud['estado']}") ⚠ ATENCIÓN REQUERIDA: Ana García OK: Luis Romero — aprobada ⚠ ATENCIÓN REQUERIDA: María López OK: Carlos Díaz — rechazada
Concepto · 20 min
El bucle while
El bucle while repite un bloque de código mientras una condición sea verdadera. A diferencia del for, no sabe de antemano cuántas veces va a iterar.
Estructura básica
while condicion: # este bloque se repite mientras condicion sea True # en algún momento, algo dentro del bloque debe cambiar # el valor de condicion para que el bucle termine
Bucle infinito: si la condición nunca se vuelve False, el programa nunca termina. Es el error más común con while. Siempre hay que asegurarse de que algo dentro del bucle modifique la condición. Para interrumpirlo: Ctrl + C en la terminal.
1
Contador simple
turnos_disponibles = 5 while turnos_disponibles > 0: print(f"Turno asignado. Quedan {turnos_disponibles - 1} disponibles.") turnos_disponibles = turnos_disponibles - 1 print("No hay más turnos disponibles para hoy.") Turno asignado. Quedan 4 disponibles. Turno asignado. Quedan 3 disponibles. Turno asignado. Quedan 2 disponibles. Turno asignado. Quedan 1 disponibles. Turno asignado. Quedan 0 disponibles. No hay más turnos disponibles para hoy.
La línea turnos_disponibles = turnos_disponibles - 1 (o equivalentemente turnos_disponibles -= 1) es la que hace que el bucle eventualmente termine.
2
Validación de entrada de usuario
Uno de los usos más frecuentes de while: seguir preguntando hasta que el usuario ingrese un valor válido.
dni_ingresado = "" while len(dni_ingresado) != 8: dni_ingresado = input("Ingrese su DNI (8 dígitos): ") if len(dni_ingresado) != 8: print("❌ DNI inválido. Debe tener exactamente 8 dígitos.") print(f"✓ DNI registrado: {dni_ingresado}")
En la terminal, input() pausa el programa y espera que el usuario escriba algo y presione Enter. Lo que escribe vuelve como texto (string).
3
Menú interactivo
El while True con break es un patrón muy usado para menús: el bucle corre indefinidamente hasta que el usuario elige salir.
while True: print("\n=== SISTEMA DE TURNOS ===") print("1. Solicitar turno") print("2. Consultar estado") print("3. Cancelar turno") print("4. Salir") opcion = input("Seleccione una opción: ") if opcion == "1": print("→ Iniciando solicitud de turno...") elif opcion == "2": print("→ Consultando estado del turno...") elif opcion == "3": print("→ Cancelando turno...") elif opcion == "4": print("Hasta luego.") break else: print("Opción inválida. Intente nuevamente.")
Concepto · 15 min
Control de flujo: break, continue y else
Python ofrece herramientas para controlar el comportamiento de los bucles con más precisión que simplemente dejarlos terminar solos.
break — Salir del bucle inmediatamente
Cuando Python encuentra un break, abandona el bucle en ese instante, sin importar si quedan elementos por recorrer.
expedientes = ["EXP-001", "EXP-002", "EXP-003 URGENTE", "EXP-004", "EXP-005"] for exp in expedientes: print(f"Revisando {exp}...") if "URGENTE" in exp: print(f"¡Expediente urgente encontrado! Deteniendo revisión.") break Revisando EXP-001... Revisando EXP-002... Revisando EXP-003 URGENTE... ¡Expediente urgente encontrado! Deteniendo revisión.
continue — Saltar al siguiente elemento
continue no termina el bucle: salta el resto del bloque para el elemento actual y pasa al siguiente.
solicitudes = [ {"id": "SOL-001", "estado": "aprobada"}, {"id": "SOL-002", "estado": "pendiente"}, {"id": "SOL-003", "estado": "aprobada"}, {"id": "SOL-004", "estado": "pendiente"}, ] print("Liquidando solicitudes aprobadas:") for sol in solicitudes: if sol["estado"] != "aprobada": continue # omitir las no aprobadas print(f" ✓ Liquidando {sol['id']}") Liquidando solicitudes aprobadas: ✓ Liquidando SOL-001 ✓ Liquidando SOL-003
Cuándo usar continue vs if/else: continue es útil para filtrar casos que no necesitan procesamiento. Hace el código más plano y fácil de leer que anidar todo dentro de un if.
else en bucles — Bloque post-iteración
Python permite agregar un else a un bucle. Ese bloque se ejecuta solo si el bucle terminó normalmente, es decir, sin un break.
dni_buscado = "28456789" ciudadanos = ["31200450", "20987654", "28456789", "40123456"] for dni in ciudadanos: if dni == dni_buscado: print(f"Ciudadano encontrado: DNI {dni}") break else: print("DNI no registrado en el sistema.") Ciudadano encontrado: DNI 28456789
Si el DNI no hubiera estado en la lista, el bucle habría terminado sin break y se habría ejecutado el else.
¿Cuándo usar for y cuándo while?
Usar FOR cuando...
✓ Recorrer una lista, tupla o string
✓ Repetir N veces exactas con range()
✓ Procesar todos los elementos de una colección
Usar WHILE cuando...
✓ No se sabe cuántas repeticiones habrá
✓ Esperar una condición externa (input del usuario)
✓ Implementar un menú o loop de aplicación

Referencia
Errores frecuentes con bucles
Los errores más comunes en esta clase y cómo resolverlos.
IndentationError: expected an indented block
Falta el bloque indentado después de for ... : o while ... :. Todo bucle necesita al menos una línea indentada debajo. Verificar que se usó Tab o 4 espacios consistentemente.
El programa no termina / se cuelga (bucle infinito)
El while tiene una condición que nunca se vuelve False. Revisar que dentro del bucle haya algo que cambie el valor de esa condición. Interrumpir con Ctrl + C en la terminal.
TypeError: 'int' object is not iterable
Se intentó usar for x in 5 en lugar de for x in range(5). El for necesita una secuencia o iterable, no un número suelto.
IndexError: list index out of range
Se accedió a una posición de lista que no existe. Las listas en Python empiezan en 0. Si la lista tiene 5 elementos, el último índice es 4, no 5. Con for x in lista esto no pasa: Python maneja los índices automáticamente.
El break no funciona "desde afuera"
break sale solo del bucle más cercano que lo contiene. Si hay bucles anidados (un for dentro de otro for), el break sale del interno, no del externo.
La variable acumuladora no da el resultado esperado
Asegurarse de inicializar la variable antes del bucle (por ejemplo total = 0). Si se inicializa dentro, se resetea en cada vuelta y el acumulador no funciona.
