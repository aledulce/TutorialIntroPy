Apertura · 10 min
¿Por qué necesitamos funciones?
En las clases anteriores escribimos código que funciona. Hoy vamos a aprender a organizarlo para que funcione a escala.
El problema: código repetido es deuda
Imaginen que el sistema de ANSES tiene que calcular multas en tres lugares distintos: al crear un turno vencido, al generar un reporte mensual y al enviar notificaciones. Sin funciones, el cálculo se copia tres veces. Si cambia la tasa de recargo, hay que encontrar y editar los tres lugares. Y siempre hay uno que se olvida.
# ❌ Sin funciones: lógica duplicada en tres partes del sistema # En la vista de turnos vencidos: recargo_turno = deuda * 0.15 total_turno = deuda + recargo_turno # En el módulo de reportes: recargo_reporte = monto * 0.15 total_reporte = monto + recargo_reporte # En las notificaciones: recargo_notif = importe * 0.15 total_notif = importe + recargo_notif # Si cambia la tasa al 18%... hay que editar 3 lugares. # En un sistema real son 30 lugares. Y alguien siempre olvida uno.
La solución: una sola fuente de verdad
Una función es un bloque de código con nombre. Se define una vez y se usa todas las veces que se necesite. Si la lógica cambia, se edita en un solo lugar y todo el sistema se actualiza.
# ✅ Con función: definir una vez, usar en todas partes def calcular_total_con_recargo(deuda, tasa=0.15): recargo = deuda * tasa return deuda + recargo # En la vista de turnos: total_turno = calcular_total_con_recargo(deuda_turno) # En el módulo de reportes: total_reporte = calcular_total_con_recargo(monto_reporte) # Si cambia la tasa, se edita solo la definición. Todo lo demás funciona.
Principio DRY: Don't Repeat Yourself. En sistemas de gestión del Estado, donde la normativa cambia con frecuencia, este principio es crítico. Una tasa que cambia por decreto debería editarse en exactamente un lugar del código.
Qué vamos a construir hoy
Al final de la clase vamos a tener un módulo tramites.py con funciones reutilizables que cualquier parte del sistema puede importar. Es el primer componente real del sistema de turnos que van a seguir construyendo en las próximas semanas.

Definir y llamar funciones
La sintaxis es simple. Lo importante es entender qué pasa en cada paso.
Anatomía de una función
def saludar_ciudadano(): # Todo lo que está indentado pertenece a la función print("Bienvenido al sistema de atención al ciudadano.") print("Por favor, tenga su DNI a mano.") # Definir la función no la ejecuta. Solo la registra. # Para ejecutarla hay que llamarla: saludar_ciudadano() Bienvenido al sistema de atención al ciudadano. Por favor, tenga su DNI a mano. # Se puede llamar todas las veces que se necesite: saludar_ciudadano() saludar_ciudadano() Bienvenido al sistema de atención al ciudadano. Por favor, tenga su DNI a mano. Bienvenido al sistema de atención al ciudadano. Por favor, tenga su DNI a mano.
La palabra clave es def, seguida del nombre, paréntesis y dos puntos. El cuerpo va indentado (4 espacios o Tab). Una función sin return ejecuta acciones pero no produce un valor.

Convenciones de nombre
Python usa snake_case para funciones: palabras en minúscula separadas por guión bajo. El nombre debe describir qué hace la función, no cómo lo hace.
# ✅ Nombres claros y descriptivos def calcular_recargo(monto): ... def validar_dni(dni): ... def registrar_turno(ciudadano): ... def enviar_confirmacion(email): ... # ❌ Nombres que no dicen nada def f(x): ... def procesar(dato): ... def hacer_cosa(): ...
Conceptos · 25 min
Parámetros, argumentos y retorno de valores
Una función sin parámetros siempre hace lo mismo. Los parámetros son lo que permite que sea flexible y reutilizable.
Parámetros básicos
# Parámetro: la variable que espera la función (en la definición) # Argumento: el valor que le pasamos (cuando la llamamos) def saludar_por_nombre(nombre): # 'nombre' es el parámetro print(f"Bienvenido/a, {nombre}.") print("Su turno fue registrado.") saludar_por_nombre("María González") # "María González" es el argumento Bienvenido/a, María González. Su turno fue registrado. saludar_por_nombre("Carlos Rodríguez") Bienvenido/a, Carlos Rodríguez. Su turno fue registrado.
La instrucción return
Muchas funciones tienen que producir un resultado, no solo hacer algo. return devuelve ese valor para que quien llamó la función pueda usarlo.
def calcular_recargo(deuda, tasa): recargo = deuda * tasa return recargo # devuelve el valor calculado # El valor retornado puede guardarse en una variable: monto_extra = calcular_recargo(85000, 0.15) print(f"Recargo: ${monto_extra}") Recargo: $12750.0 # O usarse directamente en una expresión: total = 85000 + calcular_recargo(85000, 0.15) print(f"Total a pagar: ${total}") Total a pagar: $97750.0
print() y return no son lo mismo. print() muestra algo en pantalla pero la función no devuelve nada utilizable. return produce un valor que el resto del programa puede usar, guardar o transformar.
Parámetros con valor por defecto
Cuando un parámetro casi siempre tiene el mismo valor, se puede establecer un default. Así la función es más cómoda de usar en el caso común pero sigue siendo flexible.
def calcular_total(deuda, tasa=0.15): # tasa tiene valor por defecto recargo = deuda * tasa return deuda + recargo # Caso común: usa la tasa por defecto (15%) print(calcular_total(100000)) 115000.0 # Caso especial: se sobrescribe el default print(calcular_total(100000, tasa=0.20)) 120000.0 # Útil cuando hay categorías diferentes de mora: print(calcular_total(50000, tasa=0.05)) # mora leve print(calcular_total(50000, tasa=0.30)) # mora grave 52500.0 65000.0

Ejercicio guiado — Función de validación de DNI
Los sistemas del Estado necesitan validar DNIs constantemente. Vamos a escribir una función que verifique si un DNI tiene formato válido y retorne un booleano.
def es_dni_valido(dni): # Un DNI argentino tiene entre 7 y 8 dígitos numéricos dni_str = str(dni) if not dni_str.isdigit(): return False if len(dni_str) < 7 or len(dni_str) > 8: return False return True # Probar con distintos casos: print(es_dni_valido(28456789)) # válido: 8 dígitos True print(es_dni_valido(1234567)) # válido: 7 dígitos True print(es_dni_valido(123456)) # inválido: 6 dígitos False print(es_dni_valido("AB-123456")) # inválido: no es numérico False # Usar la función en un flujo real: dni_ingresado = 28456789 if es_dni_valido(dni_ingresado): print("DNI verificado. Procediendo con el trámite.") else: print("DNI inválido. Verificar el número ingresado.")
Funciones que retornan True o False se llaman predicados. Por convención, su nombre empieza con es_, tiene_ o puede_. Hacen el código muy legible: if es_dni_valido(dni) se lee casi en castellano.

Funciones que retornan múltiples valores
Python permite retornar varios valores a la vez. Es útil cuando la función calcula varias cosas relacionadas.
def calcular_desglose(deuda, tasa=0.15): recargo = deuda * tasa total = deuda + recargo return deuda, recargo, total # retorna una tupla # Desempaquetar los tres valores: capital, mora, importe_final = calcular_desglose(85000) print(f"Capital original: ${capital:,.0f}") print(f"Mora (15%): ${mora:,.0f}") print(f"Total a pagar: ${importe_final:,.0f}") Capital original: $85,000 Mora (15%): $12,750 Total a pagar: $97,750

Scope de variables y buenas prácticas
Una de las fuentes de errores más comunes para quienes empiezan: confundir variables que existen dentro y fuera de una función.
Variables locales vs. globales
# Las variables creadas DENTRO de una función son locales. # No existen fuera de ella. def procesar_turno(): numero = 42 # variable local: solo existe aquí print(f"Turno número: {numero}") procesar_turno() Turno número: 42 print(numero) # ❌ NameError: 'numero' no existe acá NameError: name 'numero' is not defined
Esto es una feature, no un bug. Las variables locales protegen al resto del programa de cambios accidentales. Cada función es un espacio aislado.
Pasar datos correctamente: parámetros y return
La forma correcta de comunicar datos entre funciones es a través de parámetros (entrada) y return (salida). Evitar depender de variables globales.
# ❌ Mal patrón: función que depende de una variable global tasa_mora = 0.15 def calcular_mal(deuda): return deuda * tasa_mora # usa la global: frágil # ✅ Buen patrón: todo entra por parámetros def calcular_bien(deuda, tasa): return deuda * tasa # sin dependencias externas: robusto # ¿Por qué importa? Si alguien cambia tasa_mora en otro lado del # programa, calcular_mal() cambia de comportamiento sin aviso. # calcular_bien() siempre hace exactamente lo que dice.
Excepción válida para globales: constantes de configuración del sistema (nombre del organismo, versión del sistema, URL de la base de datos). Ésas sí pueden ser globales, pero solo de lectura y con nombres en MAYÚSCULAS por convención.
Docstrings: documentar funciones
En sistemas del Estado que van a ser mantenidos por otras personas (y por uno mismo en seis meses), documentar es parte del trabajo.
def calcular_total(deuda, tasa=0.15): """ Calcula el total a pagar aplicando un recargo sobre la deuda. Args: deuda (float): Monto original de la deuda en pesos. tasa (float): Tasa de recargo como decimal (ej: 0.15 = 15%). Por defecto: 0.15. Returns: float: Suma de deuda original más el recargo calculado. Ejemplo: >>> calcular_total(100000) 115000.0 >>> calcular_total(100000, tasa=0.20) 120000.0 """ recargo = deuda * tasa return deuda + recargo # El docstring es accesible desde el código: help(calcular_total) # muestra la documentación
Un buen docstring responde tres preguntas: ¿qué hace la función?, ¿qué necesita para funcionar?, ¿qué devuelve? No hace falta explicar el cómo: el código ya lo dice.

Módulos: organizar código en archivos
Un archivo Python es un módulo. Organizar las funciones en módulos es el primer paso hacia una arquitectura de software real.
¿Qué es un módulo?
Cualquier archivo .py es un módulo. Si tenemos tramites.py, podemos importar sus funciones en cualquier otro archivo del proyecto con la instrucción import. Así empezamos a construir un sistema con partes separadas y coherentes.
Paso 1 — Crear el módulo tramites.py
En la carpeta sistema-turnos, crear un archivo nuevo llamado tramites.py. Aquí van a vivir todas las funciones de negocio del sistema.
# tramites.py # Módulo de lógica de trámites para el sistema de atención al ciudadano # Semana 2 - Clase 6 TASA_MORA_DEFAULT = 0.15 # constante: mayúsculas por convención def es_dni_valido(dni): """Valida si un DNI argentino tiene formato correcto.""" dni_str = str(dni) return dni_str.isdigit() and 7 <= len(dni_str) <= 8 def calcular_total(deuda, tasa=TASA_MORA_DEFAULT): """Calcula el total a pagar con recargo por mora.""" recargo = deuda * tasa return deuda + recargo def formatear_turno(nombre, dni, fecha, hora): """Genera el texto de confirmación de un turno.""" linea = "─" * 40 confirmacion = f""" {linea} TURNO CONFIRMADO {linea} Ciudadano : {nombre} DNI : {dni} Fecha : {fecha} Hora : {hora} {linea}""" return confirmacion

Errores frecuentes con funciones
Errores que aparecen en esta clase y cómo interpretarlos.
TypeError: función_x() missing 1 required positional argument
Se llamó a la función sin pasarle todos los parámetros obligatorios. Revisar cuántos parámetros tiene la función y cuántos se están pasando. Los parámetros con valor por defecto son opcionales; los que no tienen default son obligatorios.
La función retorna None en lugar del valor esperado
Falta la instrucción return, o está mal indentada (fuera del cuerpo de la función), o hay un camino de ejecución que no llega al return. Verificar que todos los if/else terminen en un return.
NameError al intentar usar una variable definida dentro de otra función
Las variables locales solo existen dentro de su función. Para compartir un valor entre funciones, usar return y pasar el valor como parámetro. No intentar acceder a variables de otras funciones directamente.
ModuleNotFoundError al intentar importar tramites
El archivo tramites.py no está en la misma carpeta que el archivo que intenta importarlo, o la terminal no está parada en esa carpeta. Verificar con ls (Mac/Linux) o dir (Windows) que ambos archivos están en el mismo directorio.
Las pruebas del módulo se ejecutan aunque lo importe desde otro archivo
Falta el bloque if __name__ == "__main__". Todo el código de prueba debe estar dentro de ese bloque. Sin él, al importar el módulo se ejecuta todo el código suelto que haya en el archivo.
