Apertura · 10 min
¿Por qué necesitamos estructuras de datos?
Hasta ahora guardamos información en variables simples: un nombre, un número, un texto. Pero los sistemas reales manejan colecciones de datos. Esta clase aprendemos a hacerlo.
El problema de las variables simples
Imaginemos que necesitamos registrar los turnos disponibles en una oficina de ANSES. Con lo que sabemos hasta ahora haríamos esto:
# Esto no escala. ¿Qué pasa si hay 200 turnos? turno_1 = "09:00" turno_2 = "09:30" turno_3 = "10:00" turno_4 = "10:30" # ... y así hasta turno_48
No es práctico. No se puede iterar, buscar, agregar ni eliminar fácilmente. Las estructuras de datos resuelven exactamente este problema.
Las dos estructuras fundamentales de hoy
Lista []
Colección ordenada de valores
Agrupa múltiples valores en una sola variable. Mantiene el orden. Los elementos se acceden por posición (índice).
Ej: Lista de horarios disponibles, cola de turnos, documentos requeridos para un trámite.
Diccionario {}
Colección de pares clave: valor
Asocia nombres (claves) con datos (valores). Se accede por nombre, no por posición. Ideal para representar entidades.
Ej: Datos de un ciudadano: nombre, DNI, dirección, estado del trámite.
¿Dónde los vamos a usar en el proyecto?
En el sistema de turnos de ANSES que estamos construyendo:
Lista de turnos disponibles Lista de documentos requeridos Diccionario con datos del ciudadano Diccionario con datos del trámite Lista de diccionarios (tabla de turnos)
Al final de esta clase, estas estructuras ya van a ser herramientas naturales para pensar los datos del sistema.
Concepto · 30 min
Listas
Una lista agrupa múltiples valores en una sola variable. Los valores mantienen su orden y se pueden modificar.
1
Crear una lista
# Horarios disponibles en la sede Palermo de ANSES horarios = ["09:00", "09:30", "10:00", "10:30", "11:00"] print(horarios) ['09:00', '09:30', '10:00', '10:30', '11:00'] print(len(horarios)) # Cuántos elementos tiene 5
Los corchetes [] definen la lista. Los elementos van separados por comas. Pueden ser textos, números o cualquier tipo de dato.
2
Acceder a elementos por índice
Python cuenta desde cero. El primer elemento tiene índice 0, el segundo índice 1, y así.
horarios = ["09:00", "09:30", "10:00", "10:30", "11:00"] # [0] [1] [2] [3] [4] print(horarios[0]) # Primer horario 09:00 print(horarios[2]) # Tercer horario 10:00 print(horarios[-1]) # Último horario (índice negativo) 11:00
Intentar acceder a horarios[5] da IndexError: el índice 5 no existe en una lista de 5 elementos (los índices van del 0 al 4).
3
Agregar y eliminar elementos
horarios = ["09:00", "09:30", "10:00"] # Agregar al final horarios.append("10:30") print(horarios) ['09:00', '09:30', '10:00', '10:30'] # Eliminar por valor horarios.remove("09:30") # El turno fue tomado print(horarios) ['09:00', '10:00', '10:30'] # Eliminar por posición horarios.pop(0) # Elimina el primero print(horarios) ['10:00', '10:30']
4
Recorrer una lista con for
Una de las operaciones más frecuentes: hacer algo con cada elemento de la lista.
documentos_requeridos = [ "DNI original", "Constancia de CUIL", "Último recibo de sueldo", "Formulario PS.6.75" ] print("Documentos requeridos para el trámite:") for doc in documentos_requeridos: print(f" - {doc}") Documentos requeridos para el trámite: - DNI original - Constancia de CUIL - Último recibo de sueldo - Formulario PS.6.75
La variable doc es temporal: toma el valor de cada elemento en cada vuelta del loop. El nombre puede ser cualquiera, pero tiene que ser descriptivo.
5
Verificar si un elemento está en la lista
documentos_requeridos = ["DNI original", "Constancia de CUIL", "Formulario PS.6.75"] documentos_presentados = ["DNI original", "Constancia de CUIL"] for doc in documentos_requeridos: if doc in documentos_presentados: print(f" ✓ {doc}") else: print(f" ✗ {doc} — FALTA") ✓ DNI original ✓ Constancia de CUIL ✗ Formulario PS.6.75 — FALTA
El operador in devuelve True o False. Es la forma más directa de buscar en una lista en Python.
Concepto · 30 min
Diccionarios
Un diccionario asocia nombres (claves) con datos (valores). Es la estructura ideal para representar entidades del sistema: ciudadanos, trámites, turnos.
1
Crear un diccionario
# Datos de un ciudadano en el sistema ciudadano = { "nombre": "María González", "dni": 30456789, "cuil": "27-30456789-3", "cobertura_activa": True } print(ciudadano) {'nombre': 'María González', 'dni': 30456789, 'cuil': '27-30456789-3', 'cobertura_activa': True}
Las llaves {} definen el diccionario. Cada par clave: valor se separa con coma. Las claves son strings, los valores pueden ser cualquier tipo.
2
Acceder a valores por clave
ciudadano = { "nombre": "María González", "dni": 30456789, "cuil": "27-30456789-3" } print(ciudadano["nombre"]) María González print(f"DNI: {ciudadano['dni']}") DNI: 30456789 # Forma segura: .get() no lanza error si la clave no existe email = ciudadano.get("email", "sin email registrado") print(email) sin email registrado
Preferir .get(clave, valor_por_defecto) cuando no se puede garantizar que la clave exista. Evita errores KeyError en producción.
3
Agregar y modificar datos
ciudadano = { "nombre": "María González", "dni": 30456789 } # Agregar nueva clave ciudadano["telefono"] = "11-4523-8800" # Modificar valor existente ciudadano["nombre"] = "María Laura González" print(ciudadano) {'nombre': 'María Laura González', 'dni': 30456789, 'telefono': '11-4523-8800'}
4
Recorrer un diccionario
tramite = { "tipo": "Jubilación ordinaria", "expediente": "EXP-2024-00847", "estado": "En evaluación", "dias_restantes": 15 } # Recorrer claves y valores print("=== Estado del trámite ===") for clave, valor in tramite.items(): print(f" {clave}: {valor}") === Estado del trámite === tipo: Jubilación ordinaria expediente: EXP-2024-00847 estado: En evaluación dias_restantes: 15
El método .items() devuelve cada par clave-valor. Es la forma estándar de recorrer un diccionario completo.
5
Verificar si una clave existe
ciudadano = { "nombre": "María González", "dni": 30456789 } if "telefono" in ciudadano: print(f"Teléfono: {ciudadano['telefono']}") else: print("El ciudadano no tiene teléfono registrado.") El ciudadano no tiene teléfono registrado.
Usar "clave" in diccionario antes de acceder a una clave cuando no se sabe si existe. El acceso directo a una clave inexistente lanza KeyError y detiene el programa.
Integración · 20 min
Combinando listas y diccionarios
La mayoría de los datos reales son listas de diccionarios: una colección de entidades, donde cada entidad tiene sus propios campos. Esta es la base de cualquier tabla de base de datos.
Lista de diccionarios: la estructura más frecuente
Un registro de turnos del día es exactamente esto: una lista donde cada elemento es un turno (diccionario).
turnos_del_dia = [ {"horario": "09:00", "ciudadano": "Juan Martínez", "tramite": "Asignación Familiar", "estado": "atendido"}, {"horario": "09:30", "ciudadano": "Ana López", "tramite": "Jubilación", "estado": "en espera"}, {"horario": "10:00", "ciudadano": "Pedro Silva", "tramite": "Pensión", "estado": "en espera"}, {"horario": "10:30", "ciudadano": "Rosa Díaz", "tramite": "Asignación Familiar", "estado": "ausente"}, ]
Esta estructura es exactamente lo que Django va a devolver cuando consultemos la base de datos. Entenderla ahora facilita mucho el trabajo en la Semana 3.
Acceder a datos anidados
# Acceder al primer turno primer_turno = turnos_del_dia[0] print(primer_turno["ciudadano"]) Juan Martínez # En una sola línea print(turnos_del_dia[2]["tramite"]) Pensión # Recorrer y mostrar todos for turno in turnos_del_dia: print(f" {turno['horario']} {turno['ciudadano']:<20} {turno['estado']}") 09:00 Juan Martínez atendido 09:30 Ana López en espera 10:00 Pedro Silva en espera 10:30 Rosa Díaz ausente
Filtrar con condiciones
¿Cuántos turnos todavía están en espera? ¿Qué ciudadanos hay que llamar?
turnos_pendientes = [] for turno in turnos_del_dia: if turno["estado"] == "en espera": turnos_pendientes.append(turno) print(f"Turnos pendientes: {len(turnos_pendientes)}") for t in turnos_pendientes: print(f" → {t['horario']} — {t['ciudadano']}") Turnos pendientes: 2 → 09:30 — Ana López → 10:00 — Pedro Silva
Este patrón — recorrer una lista, evaluar una condición, acumular en otra lista — es uno de los más usados en backend. Lo vamos a ver cientos de veces en el curso.

Métodos útiles de listas y diccionarios
Referencia rápida de los métodos más frecuentes. No hace falta memorizarlos todos: con el uso diario se vuelven naturales.
Métodos de listas
lista = ["a", "b", "c"] lista.append("d") # Agrega al final lista.insert(1, "x") # Inserta en posición 1 lista.remove("b") # Elimina por valor (primera ocurrencia) lista.pop() # Elimina y devuelve el último lista.pop(0) # Elimina y devuelve el de posición 0 lista.sort() # Ordena en el lugar (solo para tipos comparables) lista.reverse() # Invierte el orden lista.index("a") # Devuelve la posición de "a" lista.count("a") # Cuántas veces aparece "a" lista.clear() # Vacía la lista nueva = lista.copy() # Copia la lista (no la misma referencia)
Métodos de diccionarios
d = {"a": 1, "b": 2, "c": 3} d.keys() # Lista de claves: dict_keys(['a', 'b', 'c']) d.values() # Lista de valores: dict_values([1, 2, 3]) d.items() # Lista de pares: dict_items([('a', 1), ...]) d.get("x", "no existe") # Acceso seguro con valor por defecto d.pop("b") # Elimina "b" y devuelve su valor (2) d.update({"d": 4}) # Agrega o actualiza claves en bloque d.clear() # Vacía el diccionario copia = d.copy() # Copia superficial del diccionario "a" in d # True si "a" es clave del diccionario
Errores comunes y cómo evitarlos
IndexError: list index out of range
Se intentó acceder a una posición que no existe. Verificar con len(lista) antes o usar if i < len(lista).
KeyError: 'nombre'
La clave no existe en el diccionario. Usar diccionario.get("nombre") o verificar con "nombre" in diccionario antes de acceder.
TypeError: unhashable type: 'list'
Se intentó usar una lista como clave de diccionario. Las claves deben ser tipos inmutables: strings, números, tuplas.
