Conceptos · 20 min
Variables
Una variable es un nombre que apunta a un valor guardado en memoria. Es la herramienta más básica de la programación: sin variables, no hay programa.
¿Qué es una variable?
Cuando ANSES registra un turno, necesita recordar el nombre del ciudadano, su CUIL, la fecha y la dependencia. Esos datos tienen que vivir en algún lado mientras el programa corre. Las variables son los cajones donde los guardamos.
# Definir una variable: nombre = valor nombre_ciudadano = "María González" cuil = "27-34567890-1" turno_numero = 1042 turno_disponible = True # Usar las variables print(f"Turno #{turno_numero} — {nombre_ciudadano}") Turno #1042 — María González
La variable no es el dato: es el nombre con el que Python lo recuerda. Es como la etiqueta de una carpeta en el archivo.
Reglas para nombrar variables
Python tiene reglas estrictas. Violarlas produce errores de sintaxis antes de que el programa corra.
✓ Válido
nombre_ciudadano cuil turno_numero_2 fecha_alta _id_interno
✗ Inválido
2do_turno # empieza con número nombre ciudadano # tiene espacio turno-nro # tiene guión class # palabra reservada
Convención en Python: snake_case. Todo en minúsculas, palabras separadas por guión bajo. Ej: fecha_nacimiento, codigo_dependencia. Es el estándar en todo el ecosistema Python.
Las variables se pueden reasignar
Una variable puede cambiar de valor en cualquier momento. Python usa siempre el último valor asignado.
estado_turno = "pendiente" print(estado_turno) pendiente estado_turno = "confirmado" print(estado_turno) confirmado estado_turno = "atendido" print(estado_turno) atendido
Esto modela perfectamente el ciclo de vida de un turno: cambia de estado a medida que el sistema procesa al ciudadano.
type() — inspeccionar qué hay en una variable
Python permite preguntar de qué tipo es un valor con la función type(). Muy útil para entender errores.
print(type("ANSES")) <class 'str'> print(type(42)) <class 'int'> print(type(3.14)) <class 'float'> print(type(True)) <class 'bool'>
Conceptos · 25 min
Tipos de datos
Python clasifica cada valor en un tipo. El tipo determina qué operaciones se pueden hacer con ese valor. Conocer los tipos evita la mayoría de los errores de principiante.
str
String — Texto
str
Cualquier texto: nombres, direcciones, códigos, mensajes. Siempre va entre comillas simples o dobles.
nombre = "Juan Pérez" organismo = 'AFIP' cuil = "20-12345678-3"
int
Integer — Entero
int
Números sin decimales. Contadores, identificadores, cantidades. Sin comillas.
turnos_disponibles = 48 numero_expediente = 20240123 anio = 2024
flt
Float — Decimal
float
Números con parte decimal aproximada. Útil para mediciones y porcentajes; en dinero puede generar pequeñas diferencias por precisión binaria.
tasa_recargo = 0.15 monto_multa = 12750.50 latitud = -34.6037
T/F
Boolean — Verdadero/Falso
bool
Solo dos valores posibles: True o False. Representa estados binarios: activo/inactivo, aprobado/rechazado.
turno_disponible = True cobertura_activa = False mayor_de_edad = True
Strings en profundidad — los más usados en sistemas de gestión
Los strings tienen operaciones propias que se aplican con punto. Son las más frecuentes al trabajar con datos de ciudadanos.
nombre = " María Elena González " # Eliminar espacios extra (datos de formularios siempre traen basura) print(nombre.strip()) María Elena González # Convertir a mayúsculas (para comparaciones y reportes) print(nombre.strip().upper()) MARÍA ELENA GONZÁLEZ # Longitud del string print(len(nombre.strip())) 20 # Reemplazar texto cuil = "20-12345678-3" print(cuil.replace("-", "")) 20123456783
strip(), upper(), lower(), replace() y len() aparecen constantemente al procesar datos del ciudadano. Memorizar estos cinco es prioritario.
El CUIL no es un número — ejemplo clave de por qué importa el tipo
Este es el error conceptual más común al modelar datos de personas en sistemas gubernamentales.
# ✗ Incorrecto — el CUIL como número cuil_mal = 20123456783 # ¿Qué pasa si el CUIL empieza en 01? Python elimina el cero. # ¿Cómo se muestra el guión 20-12345678-3? No se puede. # ¿Tiene sentido sumar dos CUILes? No. # ✓ Correcto — el CUIL como texto cuil_bien = "20-12345678-3" # Conserva el formato. Se puede validar, mostrar, comparar.
Regla práctica: si un número representa una identidad (DNI, CUIL, CUIT, expediente, código postal) → es str, no int. Si se va a sumar o calcular → es int o float.
Aplicación · 15 min
Float en transacciones financieras
En sistemas de cobro, facturación y liquidación, usar float puede generar diferencias de centavos. Esta sección muestra por qué y qué usar en su lugar.
Por qué no usar float para dinero
En dinero, un centavo importa. float guarda números en binario y muchos decimales no se representan exactos (por ejemplo, 0.1). Eso genera diferencias pequeñas pero reales que se acumulan en cajas, conciliaciones y reportes.
# Caso real 1: cobro de comisión por transacción comision = 0.1 + 0.2 print(comision) 0.30000000000000004 # Caso real 2: sumar 0.10 en 10.000 pagos chicos total = 0.0 for _ in range(10000): total += 0.10 print(total) 999.9999999999062
Conclusión operativa: float sirve para cálculos científicos o métricas, pero no para saldos, pagos, impuestos, intereses o facturación.
Qué usar en su lugar: Decimal o enteros
# Alternativa 1 (recomendada): Decimal para montos exactos from decimal import Decimal precio = Decimal("1250.35") impuesto = Decimal("0.21") total = precio * (1 + impuesto) print(total) 1513.9235 # Alternativa 2: enteros en centavos (muy común en pasarelas de pago) monto_centavos = 125035 # $1250.35 descuento_centavos = 5000 # $50.00 total_centavos = monto_centavos - descuento_centavos print(total_centavos) 120035
  • ✓Decimal: ideal para impuestos, intereses y redondeo contable.
  • ✓Enteros (centavos): ideal para alto volumen transaccional y APIs de pago.
  • ✓Regla práctica: guardar dinero como Decimal o centavos enteros; usar float solo para mostrar aproximaciones.

Cada tipo de dato permite distintas operaciones. Conocer cuáles son evita los TypeError que aparecen al mezclar tipos incompatibles.

Operaciones con strings
# Concatenación — unir strings con + nombre = "Juan" apellido = "Pérez" nombre_completo = nombre + " " + apellido print(nombre_completo) Juan Pérez # Repetición — multiplicar string por número separador = "=" * 40 print(separador) ======================================== # f-strings — la forma recomendada de combinar texto y variables turno = 1042 dependencia = "Palermo" print(f"Turno {turno} asignado a sede {dependencia}") Turno 1042 asignado a sede Palermo
Preferir siempre f-strings sobre concatenación con +. Son más legibles, menos propensas a error y permiten expresiones dentro de las llaves: f"Total: ${deuda * 1.15:.2f}"
Operadores de comparación — devuelven True o False
turnos_disponibles = 48 turno_solicitado = 50 print(turnos_disponibles > 0) # ¿Hay turnos? True print(turno_solicitado > turnos_disponibles) # ¿Se puede atender? False edad = 17 print(edad >= 18) # ¿Es mayor de edad? False cuil_ingresado = "20-12345678-3" cuil_sistema = "20-12345678-3" print(cuil_ingresado == cuil_sistema) # ¿Coincide el CUIL? True
Los operadores de comparación son la base de los condicionales, que vemos en la Clase 3. Todo el comportamiento lógico del sistema de turnos va a depender de estos resultados.
Conceptos · 15 min
Conversión de tipos
Los datos que ingresan al sistema (desde formularios, archivos, APIs) casi siempre llegan como texto. Saber convertirlos al tipo correcto es una habilidad fundamental.
Las funciones de conversión
# str() → convierte a texto numero = 1042 print("Turno N°: " + str(numero)) Turno N°: 1042 # int() → convierte a entero dato_formulario = "25" # llegó como texto edad = int(dato_formulario) print(edad + 1) 26 # float() → convierte a decimal monto_texto = "85000.50" monto = float(monto_texto) print(monto * 1.15) 97750.575
El error de tipo más frecuente — y cómo resolverlo
# Dato de formulario: llega siempre como string turnos_str = "48" # ✗ Esto falla total = turnos_str + 10 TypeError: can only concatenate str (not "int") to str # ✓ Convertir primero, operar después total = int(turnos_str) + 10 print(total) 58
int() falla si el string no es un número válido: int("veinte") → ValueError. En producción siempre validar antes de convertir. Lo veremos en la Semana 2 con manejo de errores.
Formatear números para salidas legibles
monto = 97750.575 # Redondear a 2 decimales print(round(monto, 2)) 97750.58 # Formatear como moneda en f-string print(f"Total a pagar: ${monto:.2f}") Total a pagar: $97750.58 # El :.2f dentro de las llaves indica "2 decimales fijos"
El formato :.2f es el estándar para mostrar montos en sistemas de gestión. Evita que aparezcan números como $97750.574999999.

Calcular disponibilidad de turnos
Un agente necesita saber cuántos turnos quedan libres en el día.
>>> capacidad_diaria = 60 >>> turnos_tomados = 47 >>> turnos_libres = capacidad_diaria - turnos_tomados >>> porcentaje_ocupacion = (turnos_tomados / capacidad_diaria) * 100 >>> print(f"Capacidad: {capacidad_diaria} turnos") >>> print(f"Tomados: {turnos_tomados}") >>> print(f"Libres: {turnos_libres}") >>> print(f"Ocupación: {porcentaje_ocupacion:.1f}%") >>> print(f"¿Hay turnos? {turnos_libres > 0}") Capacidad: 60 turnos Tomados: 47 Libres: 13 Ocupación: 78.3% ¿Hay turnos? True
3
Calcular multa con recargo
Un agente de AFIP necesita calcular el total a pagar incluyendo recargo por mora.
>>> deuda_original = 85000 >>> meses_mora = 3 >>> tasa_mensual = 0.05 # 5% mensual >>> recargo_total = deuda_original * tasa_mensual * meses_mora >>> total_a_pagar = deuda_original + recargo_total >>> print("=" * 35) >>> print(" LIQUIDACIÓN DE DEUDA — AFIP") >>> print("=" * 35) >>> print(f"Deuda original: ${deuda_original:,.2f}") >>> print(f"Meses en mora: {meses_mora}") >>> print(f"Recargo ({int(tasa_mensual*100)}%/mes): ${recargo_total:,.2f}") >>> print(f"TOTAL A PAGAR: ${total_a_pagar:,.2f}") =================================== LIQUIDACIÓN DE DEUDA — AFIP =================================== Deuda original: $85,000.00 Meses en mora: 3 Recargo (5%/mes): $12,750.00 TOTAL A PAGAR: $97,750.00
El formato :,.2f agrega separador de miles (coma) y dos decimales. Indispensable para mostrar montos en sistemas financieros del Estado.
4
Procesar datos de un formulario
Simular la llegada de datos desde un formulario web (todo llega como string) y convertirlos al tipo correcto.
# Datos tal como llegan del formulario — todo es texto >>> nombre_raw = " carlos méndez " >>> edad_raw = "34" >>> deuda_raw = "12500.75" # Limpiar y convertir >>> nombre = nombre_raw.strip().title() >>> edad = int(edad_raw) >>> deuda = float(deuda_raw) # Usar los datos ya procesados >>> print(f"Nombre: {nombre}") >>> print(f"Edad: {edad} años") >>> print(f"Deuda: ${deuda:,.2f}") >>> print(f"Mayor de edad: {edad >= 18}") Nombre: Carlos Méndez Edad: 34 años Deuda: $12,500.75 Mayor de edad: True
El método .title() capitaliza la primera letra de cada palabra — útil para normalizar nombres ingresados en mayúsculas o minúsculas.

# Causa: mezclar string y número con + print("Turno: " + 1042) # ✗ # Solución A: convertir a string print("Turno: " + str(1042)) # ✓ # Solución B: usar f-string (preferida) print(f"Turno: {1042}") # ✓

# Causa: usar variable antes de definirla, o error de tipeo print(Turno) # ✗ — Python diferencia mayúsculas print(turno) # ✗ — si no se definió antes turno = 1042 print(turno) # ✓
ValueError: invalid literal for int() with base 10: 'veinte'
# Causa: int() recibe texto que no es número edad = int("veinte") # ✗ edad = int("20") # ✓ edad = int("20.5") # ✗ — usar float() para decimales edad = int(float("20.5")) # ✓ — conversión en dos pasos

# Causa: nombre de variable inválido 2do_turno = 1042 # ✗ — empieza con número nombre completo = "Ana" # ✗ — tiene espacio segundo_turno = 1042 # ✓ nombre_completo = "Ana" # ✓
