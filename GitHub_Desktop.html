Introducción

GitHub es una plataforma de desarrollo colaborativo que permite compartir código y trabajar en equipo. GitHub Desktop es una aplicación que facilita la interacción con GitHub mediante una interfaz gráfica, eliminando la necesidad de usar comandos complejos en la terminal.

Parte 1: Creación de una cuenta de GitHub

  1. Ingresa a la página de registro de GitHub a través del siguiente enlace: https://github.com/signup
  2. Completa el formulario con:
    • Una dirección de correo electrónico válida
    • Una contraseña segura
    • Un nombre de usuario (username) único
  3. Verifica la cuenta y haz clic en el botón "Create account". Recibirás un correo de confirmación.
  4. Abre el email de confirmación y copia el código de verificación, luego pégalo en la página de GitHub.
  5. ¡Listo! Después de saltar la configuración inicial, la cuenta estará creada y lista para funcionar.

Parte 2: Instalación de GitHub Desktop

  1. Descarga GitHub Desktop desde https://desktop.github.com
    • Selecciona la versión correspondiente a tu sistema operativo (Windows, macOS o Linux)
  2. Ejecuta el archivo descargado para comenzar la instalación.
  3. Al abrir la aplicación, selecciona "Sign in to GitHub.com" e introduce tu nombre de usuario y contraseña.
    • Si ya estás logueado, la aplicación te lo mostrará y podrás saltar este paso.
  4. Haz clic en "Authorize desktop" para sincronizar la información de tu cuenta.
  5. Configura tu cuenta eligiendo "Use my GitHub account name and email address" y seleccionando tu dirección de correo electrónico.
  6. Haz clic en "Finish" para completar la instalación.

Parte 3: Conceptos básicos de Git y GitHub Desktop

Interfaz principal

La barra superior de GitHub Desktop muestra el estado actual de tu repositorio:

  • Current repository: Nombre del repositorio en el que estás trabajando
  • Current branch: La rama actual (por defecto, main)
  • Publish repository/Push origin: Botón contextual para publicar o enviar cambios

Cambios e Historia

En la barra lateral izquierda encontrarás dos vistas principales:

  • Changes: Muestra los archivos modificados que aún no has guardado (commit)
  • History: Muestra el historial de commits realizados en tu repositorio

Parte 4: Creación de un repositorio local

Opción A: Crear un repositorio completamente nuevo

  1. En GitHub Desktop, ve a File > New repository (o haz clic en "Create a New Repository on your Hard Drive")
  2. Completa los campos:
    • Name: El nombre de tu repositorio
    • Description: (Opcional) Una breve descripción del proyecto
    • Local path: Ubicación donde se guardará el repositorio en tu computadora
    • Initialize this repository with a README: Recomendado. Crea un archivo README.md
    • Git ignore: Útil para ignorar archivos específicos
    • License: Puedes agregar una licencia de código abierto
  3. Haz clic en Create repository.

Opción B: Agregar un repositorio existente

  1. Crea una carpeta en tu computadora (ejemplo: "prueba")
  2. En GitHub Desktop, ve a File > Add Local Repository
  3. Busca la carpeta y selecciona "Seleccionar carpeta"
  4. Sigue las instrucciones en pantalla para configurarlo como repositorio Git.

Parte 5: Publicar un repositorio en GitHub

  1. En la barra superior, haz clic en Publish repository
  2. En la ventana emergente:
    • Name: El nombre del repositorio (se autocompleta)
    • Description: Descripción opcional
    • Keep this code private: Desmarca esta opción para que el repositorio sea público
  3. Haz clic en Publish Repository
  4. Para ver tu repositorio en GitHub, ve a Repository > View on GitHub en el menú.

Parte 6: Trabajar con cambios (Commits)

Hacer cambios locales

  1. Abre tu repositorio en tu editor de texto preferido
  2. Realiza modificaciones en tus archivos y guárdalos
  3. En GitHub Desktop, ve a la vista Changes
  4. Selecciona con la casilla de verificación los archivos que deseas incluir en el commit
  5. En la parte inferior:
    • Escribe un resumen corto (obligatorio)
    • Opcionalmente, una descripción más detallada
  6. Haz clic en Commit to [nombre de tu rama]

Ver diferencias (Diff)

  • Líneas en verde: Texto agregado
  • Líneas en rojo: Texto eliminado
  • Cuadros amarillos: Archivos modificados

Parte 7: Sincronización con GitHub (Push y Pull)

Push: Enviar cambios locales a GitHub

  1. En la barra superior, haz clic en Push origin
  2. Los cambios ahora estarán disponibles en GitHub

Pull: Obtener cambios de GitHub

  1. Haz clic en Fetch origin para verificar si hay cambios
  2. Si hay cambios nuevos, haz clic en Pull origin
  3. Si hay conflictos, GitHub Desktop te lo notificará

Parte 8: Trabajo con Ramas (Branches)

Crear una rama nueva

  1. Ve a Branch > New Branch
  2. Escribe el nombre de la nueva rama (ejemplo: feature-nueva-funcion)
  3. Haz clic en Create Branch
  4. Para publicarla en GitHub, haz clic en Publish branch.

Cambiar entre ramas

  1. En la barra superior, haz clic en Current branch
  2. Selecciona la rama a la que deseas cambiar

Fusionar ramas (Merge)

  1. Asegúrate de estar en la rama de destino (ejemplo: main)
  2. Ve a Branch > Merge into Current Branch
  3. Selecciona la rama que deseas fusionar
  4. Haz clic en Merge [rama] into [rama_destino]
  5. Resuelve conflictos si los hay

Parte 9: Pull Requests

Crear un Pull Request desde GitHub Desktop

  1. Asegúrate de haber subido (push) los cambios de tu rama
  2. En GitHub Desktop, ve a Branch > Create Pull Request
  3. Haz clic en Create Pull Request
  4. En GitHub, escribe título y descripción, luego haz clic en Create pull request

Actualizar una rama desde main con Pull

  1. Cambia a tu rama de trabajo
  2. Ve a Branch > Update from [rama_principal]
  3. Resuelve conflictos si existen

Parte 10: Guardar cambios temporalmente (Stash)

Guardar cambios (Stash)

  1. Haz clic derecho en el encabezado de archivos modificados
  2. Selecciona Stash All Changes

Restaurar cambios guardados

  1. Asegúrate de estar en la rama correcta
  2. Ve a la vista Changes, haz clic en Stashed Changes
  3. Haz clic en Restore para recuperar los cambios

Parte 11: Clonar un repositorio existente

Clonar desde GitHub (navegador)

  1. Ve al repositorio en GitHub.com
  2. Haz clic en el botón verde Code
  3. Selecciona Open with GitHub Desktop
  4. Elige la ubicación local y haz clic en Clone.

Clonar desde GitHub Desktop

  1. Ve a File > Clone Repository
  2. Elige entre GitHub.com o URL
  3. Selecciona la ubicación y haz clic en Clone.

Parte 12: Forks y colaboración

Crear un Fork

  1. En GitHub.com, ve al repositorio que deseas bifurcar
  2. Haz clic en el botón Fork
  3. Elige nombre y haz clic en "Create Fork".

Contribuir con Pull Request desde un Fork

  1. Haz cambios en tu fork local y súbelos (push)
  2. En GitHub.com, ve a tu fork y haz clic en Contribute > Open Pull Request
  3. Crea el Pull Request

Resumen de flujo de trabajo básico

  1. Crear/Clonar repositorio
  2. Crear una rama para cambios
  3. Hacer cambios en tu editor y guardarlos
  4. Commit de los cambios localmente
  5. Push para subir cambios a GitHub
  6. Crear Pull Request para fusionar ramas
  7. Pull para sincronizar cambios de colaboradores
  8. Fusionar ramas (merge)

Solución de problemas comunes

Conflictos de merge

  1. GitHub Desktop te notificará conflictos
  2. Abre el archivo en tu editor y busca marcadores como <<<<<<< HEAD
  3. Decide qué cambios conservar
  4. Guarda y marca como resuelto en GitHub Desktop
  5. Haz commit del merge resuelto y push

No veo mis cambios en el repositorio remoto

  1. Verifica que hiciste commit localmente
  2. Asegúrate de haber hecho push
  3. Refresca la página de GitHub en tu navegador

Olvidé hacer commit antes de cambiar de rama

  1. Si GitHub Desktop te ofrece stash, elige Stash All Changes
  2. Cambia de rama y luego restaura los cambios con Restore
