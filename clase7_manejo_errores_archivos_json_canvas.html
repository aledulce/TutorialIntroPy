Concepto + práctica · 25 min
Manejo de errores con try/except
Los errores en Python son objetos llamados excepciones. Cuando un error ocurre, Python lo "lanza" (raise). Podemos "atraparlo" (except) para decidir qué hacer en lugar de dejar que el programa se detenga.
El problema: un programa frágil
Este código funciona perfectamente... hasta que el usuario ingresa algo inesperado:
# Sistema sin manejo de errores cantidad = int(input("Ingrese cantidad de turnos disponibles: ")) print(f"Hay {cantidad} turnos disponibles") # Si el usuario escribe "muchos" en lugar de un número: ValueError: invalid literal for int() with base 10: 'muchos' # El programa se cae. El sistema de atención queda fuera de servicio.
La solución: try/except
try: cantidad = int(input("Ingrese cantidad de turnos: ")) print(f"Hay {cantidad} turnos disponibles") except ValueError: print("Error: debe ingresar un número entero.") print("El sistema continuará con el valor por defecto: 0")
Regla: el bloque try contiene el código que puede fallar. El bloque except contiene qué hacer si falla. El programa sigue ejecutándose en ambos casos.
Estructura completa: try / except / else / finally
try: # Código que puede lanzar una excepción resultado = 100 / divisor except ZeroDivisionError: # Se ejecuta SOLO si ocurrió ese error específico print("Error: no se puede dividir por cero") except TypeError as e: # "as e" captura el objeto de error para inspeccionarlo print(f"Tipo de dato incorrecto: {e}") else: # Se ejecuta SOLO si NO hubo errores print(f"Resultado: {resultado}") finally: # Se ejecuta SIEMPRE, haya error o no print("Operación finalizada")
else y finally son opcionales pero muy útiles. finally se usa típicamente para cerrar archivos, conexiones de base de datos o liberar recursos —independientemente de si hubo error o no.
Ejemplo real: validar ingreso de DNI
def validar_dni(texto): """ Valida que el DNI ingresado sea un número de 7 u 8 dígitos. Retorna el DNI como entero si es válido, None si no lo es. """ try: dni = int(texto.strip()) if not (1_000_000 <= dni <= 99_999_999): raise ValueError("DNI fuera de rango válido") return dni except ValueError as e: print(f"DNI inválido: {e}") return None # Uso dni = validar_dni("28456789") # → 28456789 dni = validar_dni("abc") # → None (e imprime error) dni = validar_dni("123") # → None (fuera de rango)
raise ValueError("mensaje") permite lanzar errores propios cuando los datos no cumplen nuestras reglas de negocio, aunque Python no hubiera detectado nada malo.

Conceptos · 20 min
¿Qué es Django y para qué sirve?
Django es un framework web escrito en Python que permite construir aplicaciones web completas sin tener que programar desde cero cada componente.
Framework vs. librería: la diferencia clave
Una librería es una herramienta que se llama cuando se la necesita. Un framework es una estructura que organiza cómo se escribe el código. Con Django, la aplicación sigue las reglas del framework; el framework llama al código del desarrollador.
Analogía: una librería es como comprar ladrillos. Un framework es una casa prefabricada donde se decoran los cuartos. Django viene con paredes, techo y cañerías. Nosotros decoramos.
¿Qué incluye Django "de fábrica"?
Django sigue la filosofía batteries included: viene con todo lo necesario para construir una aplicación web de producción sin depender de librerías externas para las funciones básicas.
ORM
Interactuar con la base de datos usando Python en vez de SQL. Definís modelos como clases, Django traduce a consultas.
Panel de administración
Interfaz visual para gestionar datos sin escribir código. Lista, crea, edita y elimina registros automáticamente.
Sistema de autenticación
Usuarios, contraseñas, permisos y sesiones incluidos. No hay que programar login desde cero.
Motor de templates
Sistema para generar HTML dinámico mezclando plantillas con datos del backend.
El patrón MVT — Model, View, Template
Django organiza el código en tres responsabilidades separadas. Este patrón garantiza que el proyecto sea mantenible aunque crezca.
M
Model — Los datos
Define la estructura de la información: qué campos tiene un turno, un ciudadano, un trámite. Se mapea directamente a tablas en la base de datos.
V
View — La lógica
Funciones Python que reciben una solicitud HTTP, consultan los modelos, y devuelven una respuesta. Es el "cerebro" que conecta datos con presentación.
T
Template — La presentación
Archivos HTML con marcadores especiales que Django reemplaza con datos reales. El ciudadano ve el resultado final en el navegador.
En el sistema de turnos: el Model es el turno con fecha y estado, la View busca los turnos disponibles de mañana, y el Template los muestra como una lista en la página web.
Django en el Estado y en la industria
Django no es un framework experimental. Es uno de los más maduros del ecosistema Python, con más de 18 años de desarrollo continuo. Es usado por el Estado Nacional argentino en sistemas de gestión interna, por Instagram (que lo escala a millones de usuarios), y por plataformas de gobierno electrónico en toda la región hispanohablante. Aprender Django es aprender una habilidad directamente transferible al mercado laboral público y privado.
Setup · 15 min
Entorno virtual e instalación de Django
Antes de instalar Django, vamos a crear un entorno virtual. Esto aísla las dependencias del proyecto del resto del sistema.
¿Por qué un entorno virtual?
Si instalamos Django directamente en el sistema, todas las versiones de todos los proyectos van a competir entre sí. Un entorno virtual es una carpeta aislada con su propio Python y sus propias librerías. Es la práctica estándar en cualquier proyecto profesional.
Nunca instalar librerías con pip install sin activar antes el entorno virtual. Es uno de los errores más comunes en las primeras semanas.
Paso a paso: entorno virtual + Django
1
Ir a la carpeta del proyecto
Usar la misma carpeta sistema-turnos que creamos en la Clase 1.
Terminalcd sistema-turnos
2
Crear el entorno virtual
El nombre venv es convención. Se crea dentro de la carpeta del proyecto.
Mac/Linuxpython3 -m venv venv
Windowspython -m venv venv
3
Activar el entorno virtual
El prompt de la terminal cambia para indicar que el entorno está activo: aparece (venv) al inicio.
Mac/Linuxsource venv/bin/activate
Windowsvenv\Scripts\activate
4
Instalar Django
Con el entorno activo, pip instala Django solo dentro de ese entorno.
Instalarpip install django
Verificarpython -m django --version
Verificación exitosa
(venv) $ python -m django --version 5.x.x
Si aparece un número de versión que empieza con 4 o 5, la instalación fue correcta. Si aparece un error, revisar que el entorno esté activado ((venv) en el prompt).
Práctica guiada · 20 min
Crear el primer proyecto Django
Un proyecto Django es la unidad principal. Todo el sistema de turnos va a vivir dentro de este proyecto.
1
Crear el proyecto con django-admin
Django trae la herramienta django-admin que genera la estructura inicial del proyecto automáticamente.
(venv) $ django-admin startproject sistema_turnos . # El punto al final indica "crear el proyecto en esta carpeta" # sin crear una subcarpeta adicional — es importante
El punto (.) al final del comando es parte del comando. Sin él, Django crea una subcarpeta adicional que complica la estructura.
2
¿Qué se generó?
Después de ejecutar el comando, la carpeta tiene esta estructura:
sistema-turnos/ ├── venv/ # entorno virtual (no tocar) ├── sistema_turnos/ # configuración del proyecto │ ├── __init__.py │ ├── settings.py # ← toda la configuración │ ├── urls.py # ← rutas principales │ ├── asgi.py │ └── wsgi.py ├── manage.py # ← herramienta de comandos └── bienvenida.py # de la clase 1
3
El archivo manage.py
manage.py es el centro de operaciones del proyecto. Desde acá se ejecutan todos los comandos de Django durante el desarrollo.
# Comandos que vamos a usar a lo largo del curso: python manage.py runserver # iniciar servidor de desarrollo python manage.py startapp # crear una nueva aplicación python manage.py makemigrations # preparar cambios en la base de datos python manage.py migrate # aplicar cambios en la base de datos python manage.py createsuperuser # crear usuario administrador
Memorizar runserver y migrate: son los dos comandos que se usan en todas las sesiones de desarrollo.
Conceptos · 20 min
Estructura interna del proyecto
Entender qué hace cada archivo generado por Django. No hay que modificar nada todavía — primero leer, después escribir.
settings.py — El archivo de configuración
Contiene toda la configuración del proyecto. Hoy vamos a ver tres secciones clave.
# sistema_turnos/settings.py (fragmento) # 1. Aplicaciones instaladas INSTALLED_APPS = [ 'django.contrib.admin', # panel de administración 'django.contrib.auth', # sistema de usuarios 'django.contrib.contenttypes', 'django.contrib.sessions', 'django.contrib.messages', 'django.contrib.staticfiles', ] # 2. Base de datos (SQLite por defecto, cambiamos a PostgreSQL más adelante) DATABASES = { 'default': { 'ENGINE': 'django.db.backends.sqlite3', 'NAME': BASE_DIR / 'db.sqlite3', } } # 3. Idioma y zona horaria LANGUAGE_CODE = 'es-ar' TIME_ZONE = 'America/Argentina/Buenos_Aires'
Cambiar LANGUAGE_CODE a 'es-ar' y TIME_ZONE a 'America/Argentina/Buenos_Aires'. El panel de administración va a aparecer en español y las fechas van a estar en hora local.
urls.py — El enrutador principal
Asocia URLs con vistas. Cuando el ciudadano escribe una dirección en el navegador, Django busca en este archivo qué código ejecutar.
# sistema_turnos/urls.py from django.contrib import admin from django.urls import path urlpatterns = [ path('admin/', admin.site.urls), # Acá vamos a agregar nuestras rutas en la Clase 10 ]
Por ahora solo tiene la ruta del panel de administración. En la Clase 10 vamos a agregar las rutas del sistema de turnos.
Proyectos vs. Aplicaciones en Django
Este es uno de los conceptos que más confunde al principio. En Django hay una distinción importante:
Proyecto
sistema_turnos/
El contenedor global. Tiene la configuración, las rutas principales y conecta todo. Hay uno por sistema.
Ej: Todo el sistema de gestión de ANSES es un proyecto.
Aplicación
turnos/ ciudadanos/ tramites/
Un módulo con responsabilidad específica dentro del proyecto. Un proyecto tiene muchas aplicaciones.
Ej: El módulo de turnos, el módulo de ciudadanos y el módulo de trámites son tres aplicaciones dentro del mismo proyecto.
Esta semana trabajamos en el nivel de proyecto. En la Clase 10 vamos a crear la primera aplicación: turnos.
Práctica · 15 min
Servidor de desarrollo
Django incluye un servidor web liviano para trabajar en local. No necesitamos configurar Apache ni Nginx para desarrollar.
1
Ejecutar las migraciones iniciales
Antes de iniciar el servidor, aplicamos las migraciones que Django trae por defecto. Esto crea las tablas necesarias para el sistema de usuarios y el panel de administración.
(venv) $ python manage.py migrate Operations to perform: Apply all migrations: admin, auth, contenttypes, sessions Running migrations: Applying contenttypes.0001_initial... OK Applying auth.0001_initial... OK Applying admin.0001_initial... OK Applying sessions.0001_session... OK
Cada línea "OK" confirma que una tabla se creó correctamente en la base de datos SQLite.
2
Iniciar el servidor
(venv) $ python manage.py runserver Watching for file changes with StatReloader Performing system checks... System check identified no issues (0 silenced). September 15, 2024 - 09:30:00 Django version 5.x, using settings 'sistema_turnos.settings' Starting development server at http://127.0.0.1:8000/ Quit the server with CONTROL-C.
Abrir el navegador y entrar a http://127.0.0.1:8000/. Aparece la página de bienvenida de Django. El proyecto está corriendo.
3
Crear el superusuario
Para acceder al panel de administración necesitamos un usuario con permisos totales. Abrir una segunda terminal (con el servidor corriendo en la primera).
(venv) $ python manage.py createsuperuser Username: admin Email address: admin@sistema-turnos.gob.ar Password: •••••••• Password (again): •••••••• Superuser created successfully.
Ingresar a http://127.0.0.1:8000/admin/ con el usuario creado. El panel de administración de Django es funcional desde el primer momento, sin escribir una sola línea de HTML.
Lo que acaban de lograr
En menos de 30 minutos, sin escribir HTML ni configurar servidores, tienen un sistema web corriendo con base de datos, autenticación y panel de administración. Eso es lo que significa usar un framework maduro.

Troubleshooting
Errores frecuentes en la primera configuración de Django y cómo resolverlos.
"No module named django" al ejecutar manage.py
El entorno virtual no está activado. Verificar que el prompt tenga (venv) al inicio. Si no: source venv/bin/activate (Mac/Linux) o venv\Scripts\activate (Windows).
"That port is already in use" al ejecutar runserver
Ya hay una instancia del servidor corriendo en la terminal. Cerrar la terminal anterior o usar un puerto alternativo: python manage.py runserver 8001.
"You have 18 unapplied migration(s)" en la consola
El proyecto fue creado pero no se ejecutó migrate. Ejecutar python manage.py migrate con el entorno activo.
El panel de administración aparece en inglés
Verificar que en settings.py esté LANGUAGE_CODE = 'es-ar' (con comillas simples y guión). Guardar el archivo y recargar el navegador.
"django-admin: command not found"
El entorno virtual no está activado, o Django no se instaló dentro del entorno. Activar el entorno y ejecutar nuevamente pip install django.
El punto . en startproject generó una carpeta extra
Si la estructura quedó como sistema_turnos/sistema_turnos/ (dos niveles), se omitió el punto. La solución más limpia es eliminar la carpeta generada y ejecutar el comando de nuevo desde la carpeta correcta con el punto al final.
