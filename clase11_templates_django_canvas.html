Apertura · 10 min
¿Qué es un template?
Hasta ahora, las vistas devolvían texto plano o HTML escrito directamente en Python. Hoy separamos la lógica del diseño visual: ese es el rol de los templates.
El problema que resuelven los templates
En la Clase 8 construimos vistas que devolvían respuestas así:
# ❌ HTML incrustado en Python — difícil de mantener def lista_turnos(request): return HttpResponse("<h1>Turnos del día</h1><p>Turno 1: Juan Pérez</p>")
Esto funciona para dos líneas. Pero una pantalla real del sistema de ANSES tiene encabezado, menú, tabla de turnos, paginación, pie de página… Mezclarlo todo en Python es inmanejable.

Los templates son archivos HTML con marcadores especiales. Django los lee, reemplaza los marcadores con datos reales y devuelve HTML listo para el navegador.
La metáfora: el formulario de trámite
Pensalo como un formulario oficial en blanco: tiene estructura fija (nombre, DNI, organismo, fecha) pero los datos cambian con cada ciudadano. El template es el formulario en blanco. El contexto son los datos del ciudadano. Django los une y produce el documento final.
1
Vista obtiene datos
La función de vista consulta la base de datos o prepara los datos necesarios.
2
Vista llama al template con contexto
Le pasa un diccionario con todos los datos que el template necesita mostrar.
3
Django renderiza el template
Reemplaza cada {{ variable }} con su valor real y evalúa los {% tags %}.
4
Navegador recibe HTML completo
El ciudadano ve la pantalla con sus datos reales, como si fuera un documento personalizado.

Configuración · 15 min
Estructura de carpetas y configuración
Antes de escribir el primer template hay que indicarle a Django dónde encontrarlos.
Ubicación de los templates en el proyecto
Hay dos convenciones. En este curso usamos la que escala mejor para proyectos reales: una carpeta templates/ en la raíz del proyecto.
sistema-turnos/ ├── sistema_turnos/ # configuración del proyecto │ ├── settings.py │ ├── urls.py │ └── wsgi.py ├── turnos/ # nuestra app │ ├── views.py │ ├── urls.py │ └── models.py ├── templates/ # ← carpeta que vamos a crear hoy │ ├── base.html # template base del que heredan todos │ └── turnos/ │ ├── lista.html │ └── detalle.html └── manage.py
Registrar la carpeta en settings.py
Django no busca templates en ningún lado por defecto. Hay que decirle explícitamente dónde mirar:
# sistema_turnos/settings.py import os TEMPLATES = [ { 'BACKEND': 'django.template.backends.django.DjangoTemplates', 'DIRS': [os.path.join(BASE_DIR, 'templates')], # ← agregar esta línea 'APP_DIRS': True, 'OPTIONS': { 'context_processors': [ 'django.template.context_processors.debug', 'django.template.context_processors.request', 'django.contrib.auth.context_processors.auth', ], }, }, ]
¿Qué hace APP_DIRS: True? También busca templates dentro de la carpeta templates/ de cada app instalada. Ambos mecanismos coexisten.
Crear las carpetas
# Desde la raíz del proyecto mkdir -p templates/turnos
La carpeta templates/turnos/ usa el nombre de la app como sub-carpeta para evitar conflictos de nombres cuando el proyecto crece y tiene múltiples apps.
Conceptos · 25 min
Django Template Language (DTL)
El lenguaje de templates de Django tiene tres elementos: variables, tags y filtros. Con esos tres bloques se construye cualquier pantalla.
1. Variables — {{ variable }}
Se reemplazan por el valor real que viene del contexto de la vista. Usan notación de punto para acceder a propiedades anidadas.
<!-- templates/turnos/detalle.html --> <h1>Turno N° {{ turno.numero }}</h1> <p>Ciudadano: {{ turno.ciudadano.nombre_completo }}</p> <p>DNI: {{ turno.ciudadano.dni }}</p> <p>Fecha: {{ turno.fecha }}</p> <p>Estado: {{ turno.estado }}</p>
La notación turno.ciudadano.nombre_completo funciona con diccionarios, objetos de modelo, y listas. Django lo resuelve automáticamente.
2. Tags — {% tag %}
Lógica de control: condicionales, bucles, carga de archivos estáticos. A diferencia de las variables, algunos tags necesitan un tag de cierre.
A
Condicional — if / elif / else
{% if turno.estado == "confirmado" %} <span class="badge badge-success">Confirmado</span> {% elif turno.estado == "pendiente" %} <span class="badge badge-warning">Pendiente</span> {% else %} <span class="badge badge-danger">Cancelado</span> {% endif %}
B
Bucle — for / empty
<table> <tbody> {% for turno in turnos %} <tr> <td>{{ forloop.counter }}</td> <!-- número de iteración --> <td>{{ turno.ciudadano.nombre_completo }}</td> <td>{{ turno.hora }}</td> <td>{{ turno.estado }}</td> </tr> {% empty %} <tr><td colspan="4">No hay turnos registrados para hoy.</td></tr> {% endfor %} </tbody> </table>
{% empty %} es un bloque especial del for que se ejecuta cuando la lista está vacía. Evita el anti-patrón de if lista|length > 0.
C
Variables especiales del bucle
<!-- Variables disponibles dentro de todo bloque {% for %} --> {{ forloop.counter }} <!-- 1, 2, 3... --> {{ forloop.counter0 }} <!-- 0, 1, 2... --> {{ forloop.revcounter }} <!-- cuenta regresiva --> {{ forloop.first }} <!-- True en la primera iteración --> {{ forloop.last }} <!-- True en la última iteración -->
3. Filtros — {{ variable|filtro }}
Transforman el valor antes de mostrarlo. Se encadenan con el carácter |.
<!-- Formatear fecha --> {{ turno.fecha|date:"d/m/Y" }} <!-- 15/07/2025 --> {{ turno.fecha|date:"l j \d\e F" }} <!-- martes 15 de julio --> <!-- Texto --> {{ ciudadano.nombre|upper }} <!-- JUAN PÉREZ --> {{ ciudadano.nombre|title }} <!-- Juan Pérez --> {{ descripcion|truncatewords:20 }} <!-- corta después de 20 palabras --> {{ descripcion|default:"Sin datos" }} <!-- valor por defecto si es vacío --> <!-- Listas --> {{ turnos|length }} <!-- cantidad de elementos --> {{ turnos|first }} <!-- primer elemento --> <!-- Encadenado --> {{ ciudadano.nombre|default:"Anónimo"|upper }}
Seguridad importante: Django escapa el HTML automáticamente. Si una variable contiene <script>, lo convierte en texto inofensivo. Solo usar |safe con contenido que vos mismo controlás.
Vista que pasa datos al template
El contexto es el diccionario que une la vista con el template. Las claves del diccionario son los nombres de las variables en el template.
# turnos/views.py from django.shortcuts import render def lista_turnos(request): # Por ahora usamos datos de ejemplo (Semana 4 agregamos la BD) turnos = [ {"numero": 1, "nombre": "Juan Pérez", "hora": "09:00", "estado": "confirmado"}, {"numero": 2, "nombre": "María López", "hora": "09:30", "estado": "pendiente"}, {"numero": 3, "nombre": "Carlos Gómez", "hora": "10:00", "estado": "cancelado"}, ] contexto = { "turnos": turnos, # → accesible como {{ turnos }} en el template "organismo": "ANSES", # → {{ organismo }} "total": len(turnos), # → {{ total }} } return render(request, "turnos/lista.html", contexto)
render(request, template, contexto) es el shortcut más usado en Django. Internamente llama al motor de templates y devuelve un HttpResponse.
Conceptos · 20 min
Herencia de templates
El mecanismo más poderoso de Django para evitar repetición. Define la estructura general una sola vez y todas las páginas la heredan.
El problema sin herencia
Imaginá tener 10 páginas en el sistema de turnos: lista, detalle, nuevo turno, historial, perfil, reportes… Sin herencia, cada una repetiría el mismo encabezado, menú de navegación y pie de página. Un cambio en el logo implica editar 10 archivos.
❌ Sin herencia
lista.html → encabezado + contenido
detalle.html → encabezado + contenido
nuevo.html → encabezado + contenido
Cambiar el menú = editar N archivos
✓ Con herencia
base.html → encabezado + bloques
lista.html → solo el contenido
detalle.html → solo el contenido
Cambiar el menú = editar 1 archivo
El template base — templates/base.html
Define la estructura HTML completa. Los {% block %} son los huecos que cada página hija puede rellenar.
<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <title>{% block titulo %}Sistema de Turnos{% endblock %} — ANSES</title> <link rel="stylesheet" href="{% static 'css/estilos.css' %}"> {% block css_extra %}{% endblock %} </head> <body> <nav> <div class="nav-brand">ANSES — Sistema de Turnos</div> <ul> <li><a href="{% url 'turnos:lista' %}">Turnos</a></li> <li><a href="{% url 'turnos:nuevo' %}">Nuevo turno</a></li> </ul> </nav> <main> {% block contenido %} <!-- Cada página hija pone su contenido aquí --> {% endblock %} </main> <footer> <p>República Argentina · {{ fecha_actual|date:"Y" }}</p> </footer> {% block js_extra %}{% endblock %} </body> </html>
Template hijo — templates/turnos/lista.html
Solo declara de quién hereda y rellena los bloques. Nada más.
{% extends "base.html" %} {% block titulo %}Lista de Turnos{% endblock %} {% block contenido %} <h1>Turnos del día — {{ organismo }}</h1> <p>Total: {{ total }} turnos registrados</p> <table> <thead> <tr> <th>N°</th> <th>Ciudadano</th> <th>Hora</th> <th>Estado</th> </tr> </thead> <tbody> {% for turno in turnos %} <tr> <td>{{ turno.numero }}</td> <td>{{ turno.nombre }}</td> <td>{{ turno.hora }}</td> <td>{{ turno.estado }}</td> </tr> {% empty %} <tr><td colspan="4">Sin turnos para hoy.</td></tr> {% endfor %} </tbody> </table> {% endblock %}
Regla clave: {% extends %} debe ser siempre la primera línea del template hijo. Si hay cualquier texto antes, Django lanza un error.
Tag include — componentes reutilizables
Para fragmentos repetidos que no son páginas completas: un badge de estado, una tabla de acciones, un mensaje de alerta.
<!-- templates/componentes/badge_estado.html --> {% if estado == "confirmado" %} <span class="badge success">✓ Confirmado</span> {% elif estado == "pendiente" %} <span class="badge warning">⏳ Pendiente</span> {% else %} <span class="badge danger">✗ Cancelado</span> {% endif %} <!-- Uso en otro template --> {% include "componentes/badge_estado.html" with estado=turno.estado %}
Práctica guiada · 30 min
Construcción del sistema de templates
Vamos a construir el primer conjunto de templates del sistema de turnos. Todos los pasos se hacen en vivo.
Paso 1 — Crear el template base
Crear el archivo templates/base.html y tipear el siguiente código:
<!-- templates/base.html --> <!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1"> <title>{% block titulo %}Turnos{% endblock %} | ANSES</title> <style> body { font-family: Arial, sans-serif; margin: 0; background: #f5f5f5; } nav { background: #003876; color: white; padding: 12px 24px; display: flex; justify-content: space-between; align-items: center; } nav a { color: white; text-decoration: none; margin-left: 16px; } main { max-width: 900px; margin: 32px auto; background: white; padding: 24px; border-radius: 8px; } table { width: 100%; border-collapse: collapse; } th, td { padding: 10px 14px; border-bottom: 1px solid #eee; text-align: left; } th { background: #f8f8f8; font-size: 12px; color: #666; text-transform: uppercase; } </style> </head> <body> <nav> <span>🏛 ANSES — Sistema de Turnos</span> <div> <a href="/">Inicio</a> <a href="/turnos/">Turnos</a> </div> </nav> <main> {% block contenido %}{% endblock %} </main> <footer style="text-align:center;padding:20px;font-size:12px;color:#999"> República Argentina · Sistema de Gestión de Turnos </footer> </body> </html>
Paso 2 — Template de lista de turnos
Crear templates/turnos/lista.html:
{% extends "base.html" %} {% block titulo %}Lista de Turnos{% endblock %} {% block contenido %} <h2>Turnos del día</h2> <p style="color:#666;margin-bottom:20px"> Mostrando {{ total }} turno{{ total|pluralize }} registrado{{ total|pluralize }}. </p> <table> <thead> <tr> <th>N°</th><th>Ciudadano</th><th>Hora</th><th>Estado</th> </tr> </thead> <tbody> {% for turno in turnos %} <tr> <td>{{ forloop.counter }}</td> <td>{{ turno.nombre|title }}</td> <td>{{ turno.hora }}</td> <td> {% if turno.estado == "confirmado" %} <span style="color:green">✓ Confirmado</span> {% elif turno.estado == "pendiente" %} <span style="color:orange">⏳ Pendiente</span> {% else %} <span style="color:red">✗ Cancelado</span> {% endif %} </td> </tr> {% empty %} <tr><td colspan="4" style="text-align:center;color:#999;padding:24px"> No hay turnos registrados para hoy. </td></tr> {% endfor %} </tbody> </table> {% endblock %}
Paso 3 — Actualizar la vista
Modificar turnos/views.py para usar el template:
from django.shortcuts import render def lista_turnos(request): turnos = [ {"numero": 1, "nombre": "juan pérez", "hora": "09:00", "estado": "confirmado"}, {"numero": 2, "nombre": "MARÍA LÓPEZ", "hora": "09:30", "estado": "pendiente"}, {"numero": 3, "nombre": "carlos gómez", "hora": "10:00", "estado": "cancelado"}, ] return render(request, "turnos/lista.html", { "turnos": turnos, "total": len(turnos), "organismo": "ANSES", })
Paso 4 — Verificar
python3 manage.py runserver
Abrir http://localhost:8000/turnos/ y verificar:
Tabla con 3 turnos visibles Nombres con Title Case (filtro aplicado) Estados con color según valor Encabezado y nav del base.html
Actividad · 20 min
Actividad de cierre — Pantalla de detalle
Ahora lo hacen solos. Agregar una segunda pantalla al sistema: el detalle de un turno individual.
Tarea
Crear el template templates/turnos/detalle.html que muestre toda la información de un solo turno. La vista ya está esbozada abajo.
Vista de arranque — turnos/views.py
def detalle_turno(request, numero): # Por ahora usamos datos de ejemplo turno = { "numero": numero, "nombre": "María López", "dni": "28.456.789", "tramite": "Asignación Universal por Hijo", "hora": "09:30", "estado": "pendiente", "ventanilla": 4, "observaciones": "Traer libreta de matrimonio o partida de nacimiento.", } return render(request, "turnos/detalle.html", {"turno": turno})
Agregar la URL
# turnos/urls.py from django.urls import path from . import views urlpatterns = [ path("", views.lista_turnos, name="lista"), path("<int:numero>/", views.detalle_turno, name="detalle"), # ← nueva ]
El template que tienen que construir
El template detalle.html debe heredar de base.html y mostrar todos los campos del diccionario. Puntos extra si:
Usan if/else para el estado Usan el filtro |upper en el nombre Agregan un link "← Volver a la lista" Ocultan "Observaciones" si el campo está vacío
Para probar: ir a http://localhost:8000/turnos/1/. Si ven el detalle renderizado, funciona.

Tags y filtros más usados
Guía de consulta rápida para usar durante la práctica y entre clases.
Tags de control
<!-- Condicional --> {% if condicion %} ... {% elif otra %} ... {% else %} ... {% endif %} <!-- Bucle --> {% for item in lista %} ... {% empty %} ... {% endfor %} <!-- Herencia --> {% extends "base.html" %} {% block nombre %} ... {% endblock %} <!-- Inclusión --> {% include "componente.html" with variable=valor %} <!-- URLs inversas --> {% url 'nombre_de_la_url' argumento %} <!-- Archivos estáticos --> {% load static %} {% static 'css/estilos.css' %} <!-- Comentarios (no se renderizan) --> {# Este texto no aparece en el HTML final #}
Filtros de texto
{{ texto|upper }} <!-- MAYÚSCULAS --> {{ texto|lower }} <!-- minúsculas --> {{ texto|title }} <!-- Primer Letra En Mayúscula --> {{ texto|capfirst }} <!-- Solo primera letra --> {{ texto|truncatewords:15 }} <!-- Corta en 15 palabras... --> {{ texto|truncatechars:50 }} <!-- Corta en 50 caracteres --> {{ texto|default:"Sin datos" }} <!-- Valor por defecto si vacío --> {{ texto|length }} <!-- Longitud --> {{ texto|linebreaks }} <!-- Convierte \n en <br> --> {{ texto|pluralize }} <!-- "s" si > 1: "turno" / "turnos" -->
Filtros de fecha
{{ fecha|date:"d/m/Y" }} <!-- 15/07/2025 --> {{ fecha|date:"D d M Y" }} <!-- Tue 15 Jul 2025 --> {{ fecha|date:"l j \d\e F \d\e Y" }}<!-- martes 15 de julio de 2025 --> {{ fecha|time:"H:i" }} <!-- 09:30 --> {{ fecha|timesince }} <!-- "hace 3 horas" -->
Filtros de listas y números
{{ lista|length }} <!-- cantidad de elementos --> {{ lista|first }} <!-- primer elemento --> {{ lista|last }} <!-- último elemento --> {{ lista|join:", " }} <!-- une con separador --> {{ numero|floatformat:2 }} <!-- 2 decimales: 1234.56 -->
