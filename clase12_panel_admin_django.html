Apertura · 10 min
¿Qué es el panel de administración?
Django incluye de fábrica una herramienta que en otros frameworks tarda semanas construir. En esta clase la dejamos lista para gestionar los datos del sistema de turnos.
Una interfaz completa sin escribir una sola línea de frontend
El panel de administración de Django (/admin) es una aplicación web generada automáticamente que permite crear, leer, actualizar y eliminar registros en la base de datos. No hay que diseñar pantallas ni escribir HTML: Django lo construye a partir de los modelos que ya definimos.

En el sistema de turnos va a ser la herramienta que usan los agentes administrativos para gestionar ciudadanos, trámites y disponibilidad horaria — sin tocar la consola ni escribir SQL.
¿Para qué sirve en el mundo real?
En organismos del Estado es frecuente que el área de sistemas entregue un sistema y los empleados necesiten administrar datos sin ayuda técnica. El panel admin cubre ese rol desde el día uno.

Setup · 20 min
Activación y acceso al panel
El admin ya está incluido en Django. Solo hay que verificar la configuración y crear un superusuario para poder ingresar.
1. Verificar que django.contrib.admin esté habilitado
En settings.py, dentro de INSTALLED_APPS, estas tres líneas deben estar presentes. En un proyecto nuevo de Django ya vienen por defecto.
# sistema_turnos/settings.py INSTALLED_APPS = [ 'django.contrib.admin', # El panel en sí 'django.contrib.auth', # Sistema de usuarios y permisos 'django.contrib.contenttypes', # Necesario para permisos 'django.contrib.sessions', 'django.contrib.messages', 'django.contrib.staticfiles', 'turnos', # Nuestra app ]
2. Verificar la ruta en urls.py
La URL /admin se registra en el archivo de rutas principal del proyecto.
# sistema_turnos/urls.py from django.contrib import admin from django.urls import path urlpatterns = [ path('admin/', admin.site.urls), # Ya viene en el proyecto nuevo # ... otras rutas de la app ]
Nada más es necesario en urls.py. Django registra automáticamente todas las URLs internas del admin con esa sola línea.
3. Crear el superusuario
El superusuario tiene acceso total al panel. Se crea una sola vez por proyecto desde la terminal.
python3 manage.py createsuperuser
Nombre de usuario: admin_turnos Dirección de correo electrónico: admin@organismo.gob.ar Password: ******** Password (again): ******** Superuser created successfully.
Elegir una contraseña segura aunque sea un entorno de desarrollo. Ese hábito evita problemas cuando el proyecto pasa a producción.
4. Ejecutar el servidor y entrar al panel
python3 manage.py runserver
Abrir el navegador en http://127.0.0.1:8000/admin. Ingresar con el usuario y contraseña recién creados. El panel ya está funcionando — aunque por ahora solo muestra Usuarios y Grupos.
Práctica · 20 min
Registrar modelos en el admin
Para que los datos del sistema de turnos aparezcan en el panel, hay que registrar cada modelo. Toda la configuración vive en admin.py dentro de la app.
Modelos del sistema de turnos
Para practicar hoy vamos a usar modelos simples.
# turnos/models.py from django.db import models class Ciudadano(models.Model): nombre = models.CharField(max_length=100) apellido = models.CharField(max_length=100) dni = models.CharField(max_length=10, unique=True) email = models.EmailField(blank=True) fecha_alta = models.DateTimeField(auto_now_add=True) def __str__(self): return f"{self.apellido}, {self.nombre} — DNI {self.dni}" class Turno(models.Model): ESTADOS = [ ('pendiente', 'Pendiente'), ('confirmado', 'Confirmado'), ('cancelado', 'Cancelado'), ('atendido', 'Atendido'), ] ciudadano = models.ForeignKey(Ciudadano, on_delete=models.CASCADE) fecha_hora = models.DateTimeField() tramite = models.CharField(max_length=200) estado = models.CharField(max_length=20, choices=ESTADOS, default='pendiente') def __str__(self): return f"Turno {self.id} — {self.ciudadano} — {self.estado}"
El método __str__ define cómo aparece cada registro en el panel. Sin él, Django muestra "Ciudadano object (1)" — no es útil para nadie.
Registro básico en admin.py
El registro más simple: una línea por modelo. Es suficiente para empezar.
# turnos/admin.py from django.contrib import admin from .models import Ciudadano, Turno admin.site.register(Ciudadano) admin.site.register(Turno)
Con esto ya podemos entrar al panel y ver las secciones "Ciudadanos" y "Turnos". Podemos crear, editar y eliminar registros desde la interfaz.
Ejecutar migraciones antes de probar
Los modelos que acabamos de escribir no existen aún en la base de datos. Hay que crear y aplicar las migraciones.
python3 manage.py makemigrations python3 manage.py migrate

Configuración · 25 min
Personalización del panel
El registro básico funciona pero el panel por defecto es poco práctico para uso cotidiano. Con ModelAdmin lo adaptamos a las necesidades del organismo.
ModelAdmin: la clase de configuración
En lugar de pasar el modelo directamente a register(), creamos una clase que hereda de ModelAdmin y define cómo se muestra y comporta ese modelo en el panel.
# turnos/admin.py from django.contrib import admin from .models import Ciudadano, Turno class CiudadanoAdmin(admin.ModelAdmin): # Columnas visibles en la tabla listado list_display = ('apellido', 'nombre', 'dni', 'email', 'fecha_alta') # Campos por los que se puede buscar search_fields = ('apellido', 'nombre', 'dni') # Orden por defecto ordering = ('apellido', 'nombre') # Campos de solo lectura (no editables) readonly_fields = ('fecha_alta',) admin.site.register(Ciudadano, CiudadanoAdmin)
Personalización del modelo Turno
class TurnoAdmin(admin.ModelAdmin): list_display = ('id', 'ciudadano', 'tramite', 'fecha_hora', 'estado') list_filter = ('estado', 'fecha_hora') # Panel lateral de filtros search_fields = ('ciudadano__dni', 'ciudadano__apellido', 'tramite') date_hierarchy = 'fecha_hora' # Navegación por fecha list_per_page = 25 # Paginación list_display_links = ('id', 'ciudadano') # Columnas con enlace al detalle admin.site.register(Turno, TurnoAdmin)
La notación ciudadano__dni en search_fields permite buscar por un campo del modelo relacionado usando doble guión bajo. Es la misma sintaxis del ORM de Django.
Decorador alternativo: @admin.register
Una forma más moderna y concisa de registrar modelos. El resultado es idéntico al uso de admin.site.register().
@admin.register(Ciudadano) class CiudadanoAdmin(admin.ModelAdmin): list_display = ('apellido', 'nombre', 'dni', 'email') search_fields = ('apellido', 'nombre', 'dni') ordering = ('apellido',) @admin.register(Turno) class TurnoAdmin(admin.ModelAdmin): list_display = ('ciudadano', 'tramite', 'fecha_hora', 'estado') list_filter = ('estado',) search_fields = ('ciudadano__apellido', 'ciudadano__dni')
Ambas formas son correctas. El decorador es el estilo preferido en proyectos modernos porque mantiene el registro junto a la clase de configuración.
Columnas calculadas con métodos
Se pueden agregar columnas en list_display que no existen como campo en la base de datos, sino que calculan un valor.
@admin.register(Turno) class TurnoAdmin(admin.ModelAdmin): list_display = ('ciudadano', 'tramite', 'fecha_hora', 'estado', 'esta_pendiente') def esta_pendiente(self, obj): return obj.estado == 'pendiente' esta_pendiente.boolean = True # Muestra ícono ✓ o ✗ esta_pendiente.short_description = '¿Pendiente?' # Título de columna
Funcionalidad avanzada · 20 min
Acciones en lote y filtros
El panel permite ejecutar operaciones sobre múltiples registros seleccionados. Es útil para cambios masivos de estado, como confirmar o cancelar varios turnos a la vez.
Acciones personalizadas
El panel ya tiene la acción "Eliminar" por defecto. Podemos agregar las nuestras para operaciones frecuentes del organismo.
@admin.register(Turno) class TurnoAdmin(admin.ModelAdmin): list_display = ('ciudadano', 'tramite', 'fecha_hora', 'estado') list_filter = ('estado',) actions = ['confirmar_turnos', 'cancelar_turnos'] def confirmar_turnos(self, request, queryset): """Confirma todos los turnos seleccionados.""" actualizados = queryset.update(estado='confirmado') self.message_user( request, f"{actualizados} turno(s) confirmados correctamente." ) confirmar_turnos.short_description = "✓ Confirmar turnos seleccionados" def cancelar_turnos(self, request, queryset): """Cancela todos los turnos seleccionados.""" actualizados = queryset.update(estado='cancelado') self.message_user( request, f"{actualizados} turno(s) cancelados.", level='WARNING' ) cancelar_turnos.short_description = "✗ Cancelar turnos seleccionados"
El parámetro queryset contiene exactamente los objetos seleccionados en la lista. Todo lo que se puede hacer con un queryset de Django es válido dentro de una acción.
Filtros en el panel lateral
list_filter acepta campos del modelo y también clases de filtro personalizadas para lógica más compleja.
from django.utils import timezone class TurnosPorFecha(admin.SimpleListFilter): title = "período" parameter_name = "periodo" def lookups(self, request, model_admin): return [ ('hoy', 'Hoy'), ('semana', 'Esta semana'), ('mes', 'Este mes'), ] def queryset(self, request, queryset): hoy = timezone.now().date() if self.value() == 'hoy': return queryset.filter(fecha_hora__date=hoy) if self.value() == 'semana': return queryset.filter(fecha_hora__week=hoy.isocalendar().week) if self.value() == 'mes': return queryset.filter(fecha_hora__month=hoy.month) @admin.register(Turno) class TurnoAdmin(admin.ModelAdmin): list_filter = ('estado', TurnosPorFecha)
Personalizar el título del sitio
Por defecto el panel muestra "Administración de Django". Se puede cambiar para reflejar el organismo.
# turnos/admin.py (al final del archivo) admin.site.site_header = "ANSES — Sistema de Gestión de Turnos" admin.site.site_title = "ANSES Admin" admin.site.index_title = "Panel de administración"
Práctica guiada · 20 min
Construcción del admin completo del sistema de turnos
Integramos todo lo visto en una sola práctica. Al final, el panel va a estar listo para que un agente administrativo pueda operar el sistema sin conocimientos técnicos.
Paso 1 — admin.py completo
Tipear el código, no copiarlo. Es la forma más rápida de internalizar la sintaxis.
# turnos/admin.py from django.contrib import admin from django.utils import timezone from .models import Ciudadano, Turno # ── Ciudadano ────────────────────────────────────────────── @admin.register(Ciudadano) class CiudadanoAdmin(admin.ModelAdmin): list_display = ('apellido', 'nombre', 'dni', 'email', 'fecha_alta') search_fields = ('apellido', 'nombre', 'dni') ordering = ('apellido', 'nombre') readonly_fields = ('fecha_alta',) list_per_page = 30 # ── Turno ────────────────────────────────────────────────── class FiltroPeríodo(admin.SimpleListFilter): title = "período" parameter_name = "periodo" def lookups(self, request, model_admin): return [('hoy', 'Hoy'), ('semana', 'Esta semana')] def queryset(self, request, queryset): hoy = timezone.now().date() if self.value() == 'hoy': return queryset.filter(fecha_hora__date=hoy) if self.value() == 'semana': inicio = hoy - timezone.timedelta(days=hoy.weekday()) return queryset.filter(fecha_hora__date__gte=inicio) @admin.register(Turno) class TurnoAdmin(admin.ModelAdmin): list_display = ('id', 'ciudadano', 'tramite', 'fecha_hora', 'estado', 'esta_pendiente') list_filter = ('estado', FiltroPeríodo) search_fields = ('ciudadano__apellido', 'ciudadano__dni', 'tramite') date_hierarchy = 'fecha_hora' list_per_page = 25 list_display_links = ('id', 'ciudadano') actions = ['confirmar_turnos', 'cancelar_turnos'] def esta_pendiente(self, obj): return obj.estado == 'pendiente' esta_pendiente.boolean = True esta_pendiente.short_description = 'Pendiente' def confirmar_turnos(self, request, queryset): n = queryset.update(estado='confirmado') self.message_user(request, f"{n} turno(s) confirmados.") confirmar_turnos.short_description = "✓ Confirmar seleccionados" def cancelar_turnos(self, request, queryset): n = queryset.update(estado='cancelado') self.message_user(request, f"{n} turno(s) cancelados.", 'WARNING') cancelar_turnos.short_description = "✗ Cancelar seleccionados" # ── Identidad del sitio ──────────────────────────────────── admin.site.site_header = "ANSES — Gestión de Turnos" admin.site.site_title = "ANSES Admin" admin.site.index_title = "Panel de administración"
Paso 2 — Cargar datos de prueba desde el panel
Entrar al panel en /admin y crear manualmente:
1
Crear 3 ciudadanos
Ir a Ciudadanos → Agregar ciudadano. Completar nombre, apellido y DNI. Verificar que aparecen con el formato del __str__ definido.
2
Crear 5 turnos
Ir a Turnos → Agregar turno. El campo ciudadano debería mostrar un desplegable con los ciudadanos creados. Asignar distintos estados.
3
Probar la acción en lote
Seleccionar 2 turnos pendientes con el checkbox, elegir "Confirmar seleccionados" en el desplegable de acciones y ejecutar. Verificar el mensaje de confirmación.
4
Probar la búsqueda y los filtros
Buscar por DNI. Filtrar por estado. Verificar que el filtro de período "Hoy" funciona.
